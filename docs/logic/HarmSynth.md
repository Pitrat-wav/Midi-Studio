# 📘 АУДИТ: HarmSynth.ts

## 1. Глобальное Описание

### Назначение
Этот файл реализует самый сложный инструмент в приложении — **HarmSynth**. Это гибридный модульный синтезатор, объединяющий классический субтрактивный синтез (3 осциллятора + шум + фильтры) с методами "West Coast" (в стиле Buchla 259). Он поддерживает FM-синтез, амплитудную модуляцию и волновое сложение (Wavefolding).

### Связи
- **Импорты:** `tone` (аудио-движок).
- **Используется в:** `src/store/audioStore.ts` и управляется в `src/components/HUD/HarmScreen.tsx`.
- **Архитектура:** Использует пул голосов (`Map<string, HarmVoice>`) для обеспечения 4-голосной полифонии без лишней нагрузки на процессор.

---

## 2. Интерфейс и UI Элементы

Этот файл управляет огромным количеством параметров, которые вынесены в UI:
- **Oscillator Rack:** Выбор формы волны (Saw, Square, Tri, Sine) для 3-х источников.
- **Envelope Filters:** ADSR-огибающие для каждого осциллятора отдельно.
- **FX Rack:** Встроенные эффекты Distortion, Phaser, Chorus, Delay, Reverb.
- **Complex Section:** Секция управления West Coast синтезом (Timbre, Fold, FM Index).

## 3. Логика и Математика

### 3.1 Ядро голоса (HarmVoice)

**Строки:** 30-216

Каждый голос имеет собственную цепочку:
$$Osc(1..3) \rightarrow Env \rightarrow [Wavefolder] \rightarrow FX\_Send/Direct\_Bus$$

---

### 3.2 Алгоритм Wavefolding (Складывание волны)

**Строки:** 99-108, 344-368

Это математическое ядро "West Coast" звука. Вместо того чтобы отсекать частоты фильтром, алгоритм "загибает" пики волны обратно, создавая новые гармоники.

**Математическая модель:**

$$f(x) = \begin{cases} 
2 - x & \text{если } x > 1 \\
-2 - x & \text{если } x < -1 \\
x & \text{иначе}
\end{cases}$$

Эта операция повторяется рекурсивно несколько раз (параметр `order`).

**Построчный разбор (Функция `updateWavefolder`):**
1. **timbre:** Увеличивает входное усиление (gain), заставляя волну чаще "складываться" об границы [-1, 1].
2. **harmonics (symmetry):** Добавляет смещение постоянного тока (DC Offset), что делает складывание асимметричным, добавляя четные гармоники.
3. **Цикл (строки 357-365):** Применяет складывание по стадиям, позволяя плавно переходить от чистого синуса к сложному спектру.

---

### 3.3 FM и AM Модуляция (Buchla Mode)

**Строки:** 134-161

Когда включен `complexMode`, **Осциллятор 2** становится модулятором для **Осциллятора 1**.

**Математические модели:**
- **Частотная модуляция (FM):**
  $$Freq_{Carrier} = Freq_{Base} + (Osc2_{output} \times FM\_Index)$$
- **Амплитудная модуляция (AM):**
  $$Amp_{final} = Osc1_{output} \times (1.0 + Osc2_{output} \times AM\_Index)$$

---

### 3.4 Глобальный FX Rack

**Строки:** 272-290

Синтезатор имеет внутреннюю шину эффектов, которая работает параллельно с прямой шиной.
- **Direct Bus:** Чистый сигнал проходит через два последовательных фильтра.
- **FX Bus:** Сигнал проходит через цепочку `Distortion -> Phaser -> Chorus -> Delay -> Reverb`.
