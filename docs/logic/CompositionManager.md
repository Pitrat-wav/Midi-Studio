# CompositionManager.ts

### 1. Глобальное Описание (File Overview)
* **Назначение:** Логический мост между детерминированным движком генерации композиций (Python/Pyodide) и React-сторами инструментов. Служит для применения сгенерированных паттернов к аудио-движку.
* **Связи:** 
    * Импортирует: `useDeterministicEngine` (хук воркера Pyodide), `useBassStore`, `useDrumStore`.
    * Используется в: `App.tsx` (хук вызывается на верхнем уровне).

### 2. Интерфейс и API (Core Logic API)

* **Метод: `requestNewComp(scale, root)`**
    * **Логика:** Инициирует генерацию новой последовательности нот через воркер.

### 3. Логика и Математика (Logic & Math)

* **Функция: `useCompositionManager` (useEffect)**
    * **Что делает:** Следит за появлением новых результатов из AI/Deterministic движка и распределяет их по инструментам.

* **Математическая модель (Pattern Conversion):**
    Преобразование сырого бинарного массива из Python в формат `BassStep`.
    $$ active_i = (lastResult.bass_i == 1) $$
    Все остальные параметры (note, velocity, slide) устанавливаются в дефолтные значения.

* **Построчный разбор:**
    1. Проверяем наличие `lastResult`.
    2. Если есть секция `bass`:
        * Проходим циклом `.map` по массиву.
        * Формируем объекты со свойствами `active: true/false`, `note: 'C1'`.
        * Вызываем `bass.setPattern`.
    3. Вызов `requestNewComp` передает параметры тональности в `generatePattern`.
