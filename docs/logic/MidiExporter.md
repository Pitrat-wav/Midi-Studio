# 📘 АУДИТ: MidiExporter.ts

## 1. Глобальное Описание

### Назначение
Этот файл отвечает за экспорт созданной музыки в профессиональный формат MIDI. Он берет внутренние структуры данных приложения (паттерны барабанов, шаги баса, последовательности Тьюринга) и упаковывает их в стандартный бинарный MIDI-файл, который можно открыть в любой DAW (Ableton, FL Studio, Logic Pro).

### Связи
- **Библиотеки:** Использует `midi-writer-js` для формирования файлов.
- **Данные:** Получает паттерны напрямую из `instrumentStore.ts` и `StingGenerator.ts`.
- **Использование:** Вызывается при нажатии кнопки "Export MIDI" в различных модулях студии.

## 3. Логика и Математика

### 3.1 Функция: `exportToMidi(...)`

**Строки:** 6-229

**Что делает:**  
Принимает параметры текущего состояния инструментов и генерирует `Uint8Array` (бинарные данные MIDI-файла).

**Логические блоки по инструментам:**

#### 1. Drums (Барабаны) — Строки 46-68
Транслирует сетку 16x16 в MIDI-ноты:
- `Kick` → Нота `C1`.
- `Snare` → Нота `D1`.
- `Hi-Hat` → Нота `F#1`.
- `Clap` → Нота `D#1`.

Все события записываются как 1/16 доли (duration: "16").

#### 2. Bass (Бас) — Строки 23-43
Преобразует "кислотные" параметры в MIDI-атрибуты:
- **Акценты:** Если `step.accent` включен → `velocity: 127`.
- **Слайды (Slides):** Если на шаге есть слайд, длительность ноты увеличивается до 1/8, чтобы создать наложение нот (Overlap), которое в DAW интерпретируется как Portamento/Legato.

#### 3. ML-185 Sequencer — Строки 72-110
Рассчитывает длительность нот на основе параметра `pulseCount` (повторения):
$$Duration_{MIDI} = StageLength \times PulseCount$$

### 3.2 Эмуляция последовательностей Тьюринга

**Строки:** 197-225

Поскольку паттерн Тьюринга — это состояние битового регистра, а не фиксированный массив, экспортер имитирует работу алгоритма на 64 шага вперед, используя ту же математику сдвига, что и `TuringMachine.ts`.

**Математическая модель сдвига:**
$$bit = (reg \gg (bits-1)) \land 1$$
$$reg_{next} = ((reg \ll 1) \lor randomBit) \land Mask$$

Это позволяет экспортировать "будущее" развитие мелодии, исходя из текущего состояния.

---

### 3.3 Преобразование частоты в ноту MIDI

Используется для перевода числовых значений (Pitch) в строковые имена нот:
$$MidiNumber = 48 + (Register \pmod{16})$$
$$NoteName = Tone.Frequency(MidiNumber, 'midi').toNote()$$
