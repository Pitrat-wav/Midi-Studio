# 📘 АУДИТ: AudioTrackManager.ts

## 1. Глобальное Описание

### Назначение
`AudioTrackManager` — это синглтон-сервис, отвечающий за синхронизацию визуальных аудио-клипов с реальным аудио-движком. Он управляет "варпингом" (временным растяжением), кэшированием плееров и динамическим изменением скорости воспроизведения при изменении BPM проекта.

### Связи
- **Stores:** Взаимодействует с `audioStore` для регистрации плееров и `arrangementStore` для получения данных о клипах.
- **Tone.js:** Использует `Tone.Player` и `Tone.GrainPlayer`.

---

## 2. Интерфейс и Логика

### Основные методы:
- **`createPlayer(clip)`**: Асинхронно создает узел воспроизведения. Если режим `Stretch`, создается `GrainPlayer`.
- **`updatePlaybackRate(player, clip, bpm)`**: Рассчитывает коэффициент ускорения/замедления.
- **`onBpmChange(bpm, clips)`**: Глобальный слушатель, обновляющий все активные плееры.

---

## 3. Логика и Математика

### 3.1 Алгоритм Варпинга

**Математическая модель:**
Коэффициент скорости воспроизведения ($$Rate$$) рассчитывается как отношение текущего BPM проекта к оригинальному BPM файла:
$$PlaybackRate = \frac{BPM_{current}}{BPM_{original}}$$

### 3.2 Режимы обработки (Warp Modes)
1. **Stretch**: Использует алгоритм Granular Synthesis. Позволяет менять скорость без изменения высоты тона (Pitch-shifting).
2. **Repitch**: Классический метод. При ускорении звук становится выше, как на виниловой пластинке.
3. **None**: Воспроизведение "как есть". Полезно для звуковых эффектов (One-shots), не привязанных к темпу.
