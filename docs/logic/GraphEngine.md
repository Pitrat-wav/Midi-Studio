# 📘 АУДИТ: GraphEngine.ts

## 1. Глобальное Описание

### Назначение
Этот файл — "сердце" модульной системы приложения. **GraphEngine** (Движок Графа) отвечает за превращение визуальной схемы узлов (Node Editor) в реальную сеть обработки звука. Он управляет жизненным циклом аудио-модулей, их соединением и выполнением пользовательских JavaScript-скриптов внутри аудио-потока. Это позволяет пользователю "собирать" собственные синтезаторы и логические схемы прямо в браузере.

### Связи
- **Импорты:** `tone`, `reactflow`, `nodeStore`.
- **Используется в:** Редакторе узлов (`NODES` view).
- **Архитектура:** Статический класс-менеджер, который подписывается на изменения магазина `nodeStore` и синхронизирует визуальное состояние с аудио-движком.

---

## 2. Интерфейс и UI Элементы

### Типы узлов (NodeType)
Движок поддерживает десятки типов модулей:
- **Audio:** Осцилляторы, Фильтры, LFO, Delay, Reverb, VCA.
- **Instruments:** Предварительно настроенные Kick, Snare, Hat.
- **Logic:** Сложение/Умножение сигналов, Сравнение, Случайные числа.
- **IO:** Вход MIDI, Выход на динамики.
- **Script:** Поле для написания собственного DSP кода на JavaScript.

## 3. Логика и Математика

### 3.1 Фабрика узлов (`createNode`)

**Строки:** 93-704

Этот метод — гигантский `switch-case`, который по типу узла создает соответствующий объект `Tone.js`.
При создании каждого узла формируется **Wrapper** (обертка), которая сопоставляет визуальные "ручки" (Handles) на схеме с реальными параметрами звука (`AudioParam`).

**Пример (Осциллятор):**
- Входной хендл `fm` → Мапится на `oscillator.frequency`.
- Входной хендл `detune` → Мапится на `oscillator.detune`.

---

### 3.2 Пользовательские скрипты (`createScriptNode`)

**Строки:** 61-91

Это уникальная фича, позволяющая писать код, который выполняется непосредственно процессором (ScriptProcessorNode).

**Математическая модель (Audio Processing):**
Код пользователя оборачивается в функцию, которая получает массив входящих сэмплов и должна заполнить массив исходящих:
$$Output[i] = Logic(Input[i], Memory)$$

---

### 3.3 Математика логических узлов

#### Euclidean Sequencer (Эвклидов секвенсор в графе)
**Строки:** 540-621
Реализует алгоритм Брезенхэма для распределения ударов.
**Формула хита:**
$$IsHit = ((Phase + Rotate) \times Pulses) \pmod{Steps} < Pulses$$
Это позволяет создавать сложные ритмы внутри модульной сетки.

#### ADSR Envelope (Огибающая)
**Строки:** 381-414
Реализует конечный автомат (State Machine) для уровней громкости:
1. **Attack:** Линейное нарастание $level += 1 / AttackTime$.
2. **Decay:** Спад до уровня Sustain.
3. **Sustain:** Удержание уровня, пока сигнал Gate > 0.5.
4. **Release:** Экспоненциальное затухание $level \times 0.999$.

---

### 3.4 Перестроение соединений (`rebuildConnections`)

**Строки:** 767-825

Когда пользователь рисует линию между узлами, движок:
1. Отключает все старые связи (`disconnect`).
2. Для каждой линии находит исходный узел и целевой параметр.
3. Вызывает команду `source.connect(destination)`.

Это обеспечивает мгновенную реакцию звука на изменение схемы.
