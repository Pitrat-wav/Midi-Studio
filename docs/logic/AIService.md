# 📘 АУДИТ: AIService.ts

## 1. Глобальное Описание

### Назначение
Этот файл представляет собой сервисный слой для интеграции алгоритмов искусственного интеллекта в рабочее пространство студии. Он служит интерфейсом для запроса генеративного контента: от визуальных текстур для 3D объектов до логических советов по сведению звука. 

### Связи
- **Используется в:** `src/store/aiStore.ts` для управления запросами.
- **Интерфейс:** Связан с панелью **AI Texture Lab** и HUD-сообщениями.
- **Статус:** В текущей версии реализован как прокси-сервис с заглушками (mocks), готовый к подключению внешних API (Stable Diffusion, GPT-4, Hugging Face).

---

## 2. Интерфейс и UI Элементы

### Элемент: Генерация текстур (generateTexture)
* **Вход:** Текстовый промпт от пользователя.
* **Результат:** URL-ссылка на изображение, которое мгновенно накладывается на 3D-модели в сцене.

### Элемент: AI Агент (chatAgent)
* **Вход:** Контекст текущего состояния студии.
* **Результат:** Текстовый ответ, который может отображаться в статус-баре или чате.

---

## 3. Логика и Математика

### 3.1 Метод `generateTexture(prompt)`

**Строки:** 2-17

**Что делает:**  
Имитирует работу нейросети Stable Diffusion. Принимает описание стиля (например, "Cyberpunk nebula") и возвращает путь к сгенерированному файлу.

**Логика:**
- Использует асинхронный вызов (`async/await`) для имитации ожидания ответа от сервера (2000мс).
- Выбирает случайный ассет из заранее подготовленного пула `mockTextures` для демонстрации работы.

---

### 3.2 Метод `chatAgent(context)`

**Строки:** 19-33

**Что делает:**  
Имитирует работу LLM-ассистента, который анализирует, что делает пользователь.

**Логика:**
- Принимает `context` (например, текущий BPM или настройки фильтров).
- Возвращает одну из экспертных фраз, симулируя "понимание" музыкального процесса (например, "Я оптимизировал детюн осциллятора для более плотного звука").
