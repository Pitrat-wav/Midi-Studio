# 📘 АУДИТ: PadGenerator.ts

## 1. Глобальное Описание

### Назначение
Этот файл реализует алгоритм генерации гармонических последовательностей для эмбиент-пэдов. **PadGenerator** берет текущую тональность и лад проекта и автоматически строит последовательность аккордов (I-IV-V-I), которые звучат гармонично и создают основу для трека. Уровень сложности (Complexity) определяет, насколько насыщенными будут аккорды.

### Связи
- **Библиотеки:** Использует `@tonaljs/tonal` для работы с музыкальными интервалами.
- **Используется в:** `src/store/audioStore.ts` для автоматической смены нот в пэд-синтезаторе.
- **Влияние:** Напрямую зависит от настроек в `useHarmonyStore`.

## 2. Интерфейс и UI Элементы

Модуль управляется следующими параметрами в HUD:
- **Complexity (Сложность):** Если значение ручки > 0.6, генератор добавляет в аккорды септиму (7-ю ступень) в более высокой октаве, делая звук более "джазовым" или воздушным.

## 3. Логика и Математика

### 3.1 Алгоритм прогрессии

**Строки:** 11-12

По умолчанию используется классическая музыкальная прогрессия:
- **I** (Тоника) — 0 ступень.
- **IV** (Субдоминанта) — 3 ступень.
- **V** (Доминанта) — 4 ступень.

### 3.2 Построение аккорда

**Строки:** 14-25

Для каждого шага прогрессии строится аккорд, состоящий исключительно из нот выбранного лада.

**Математическая модель:**
Для ступени $D$ аккорд строится по индексам лада:
$$Chord = [Notes[D \pmod N], Notes[(D+2) \pmod N], Notes[(D+4) \pmod N]]$$

Где $N$ — количество нот в ладу (обычно 7).

**Построчный разбор:**
1. Определяет корень аккорда (`chordRoot`) в 3-й октаве.
2. Добавляет терцию (`third`) и квинту (`fifth`), отсчитывая по 2 ступени гаммы.
3. **Проверка сложности:** Если `complexity > 0.6` → добавляется 7-я ступень (`degree + 6`) в 4-й октаве.
