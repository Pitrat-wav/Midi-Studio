# 📘 АУДИТ: StingGenerator.ts

## 1. ГЛОБАЛЬНОЕ ОПИСАНИЕ

### Назначение
Этот файл реализует **генератор басовых линий в стиле "Sting by Iftah"** — классический паттерн-генератор для TB-303 Acid Bass. Алгоритм создаёт вероятностные паттерны из 16 шагов, где каждый шаг может быть активным/неактивным, иметь слайд (портаменто) и акцент. Это ядро "кислотного" звучания жанров Acid Techno и House.

### Связи

  - `@tonaljs/tonal` → Библиотека музыкальной теории для работы с ладами и нотами.

## 3. ЛОГИКА И МАТЕМАТИКА

### 3.1 Интерфейс: `BassStep` (Строки 4-10)

**Что делает:** Описывает один шаг паттерна баса (1/16 доля).

**Поля:**
- `note: string` → Название ноты с октавой (например, `"C2"`, `"D#3"`).
- `velocity: number` → Громкость (0.7 обычная, 1.0 с акцентом).
- `slide: boolean` → Включён ли портаменто (скольжение к следующей ноте).
- `accent: boolean` → Усилен ли удар (влияет на фильтр и громкость).
- `active: boolean` → Звучит ли нота на этом шаге.

---

### 3.2 Функция: `generateBassPattern(...): BassStep[]`

**Строки:** 17-90

**Что делает:**  
Генерирует массив из 16 шагов басового паттерна согласно параметрам `density` и `type`.

**Параметры:**
- `density` ($D \in [0, 1]$) → Плотность паттерна. При $D = 0$ все шаги пусты, при $D = 1$ все активны.
- `type` ($T \in [0, 1]$) → Характер мелодии:
  - $T < 0.3$ → Тональный бас (Root, 3rd, 5th степени).
  - $0.3 \leq T < 0.7$ → Диатонический (все ступени лада).
  - $T \geq 0.7$ → Хроматический (все 12 нот).
- `root` → Тональный центр (например, `"C"`, `"F#"`).
- `scale` → Тип лада (`'minor'`, `'major'`, `'dorian'` и т.д.).
- `octave` → Октава баса (обычно 1 или 2).
- `seedA`, `seedB` → Семена псевдослучайного генератора.
- `morph` ($M \in [0, 1]$) → Интерполяция между seedA (при $M=0$) и seedB (при $M=1$).

---

**Математическая модель:**

#### 1. Выбор Scale Notes (Строки 29-43)

$$
ScaleNotes =
\begin{cases}
[Root, 3rd, 5th] & \text{if } T < 0.3 \\
FullScale & \text{if } 0.3 \leq T < 0.7 \\
ChromaticScale & \text{if } T \geq 0.7
\end{cases}
$$

**Построчный разбор:**

**Строка 31:** Получаем полный лад из библиотеки Tonal.js:
```typescript
const scaleObj = Scale.get(`${root} ${scale}`)
```
Например, для `"C minor"` вернёт: `["C", "D", "D#", "F", "G", "G#", "A#"]`.

**Строки 34-35:** Если `type < 0.3` → берём только триаду (индексы 0, 2, 4):
```typescript
scaleNotes = fullScale.filter((_, i) => [0, 2, 4].includes(i))
```
Для C-minor: `["C", "D#", "G"]` (тоника, малая терция, квинта).

**Строки 36-37:** Если `0.3 ≤ type < 0.7` → используем весь лад.

**Строки 38-40:** Если `type ≥ 0.7` → хроматическая гамма (все 12 нот):
```typescript
const chromaticScale = Scale.get(`${root} chromatic`)
```

---

#### 2. Псевдослучайный генератор весов (Строки 46-49)

**Функция:** `getWeight(i: number, s: number): number`

$$
W(i, s) = \text{frac}(\sin(s + i \times 13.37) \times 10000)
$$

Где $\text{frac}(x) = x - \lfloor x \rfloor$ (дробная часть).

**Построчный разбор:**

**Строка 47:** Детерминированная псевдослучайность через синус:
```typescript
const x = Math.sin(s + i * 13.37) * 10000
```
- `s` → seed (семя).
- `i` → индекс шага.
- `13.37` → магическая константа для дисперсии.

**Строка 48:** Извлекаем дробную часть (возвращаем число от 0 до 1):
```typescript
return x - Math.floor(x)
```

---

#### 3. Цикл генерации 16 шагов (Строки 51-87)

Для каждого шага $i \in [0, 15]$:

##### a) Вычисление активности (Строки 52-57)

$$
W_i = W_A(i) \times (1 - M) + W_B(i) \times M
$$

$$
active_i = \begin{cases}
true & \text{if } W_i < D \\
false & \text{otherwise}
\end{cases}
$$

**Построчный разбор:**

**Строки 53-54:** Получаем веса для обоих семян:
```typescript
const weightA = getWeight(i, seedA)
const weightB = getWeight(i, seedB)
```

**Строка 55:** Морфинг (плавная интерполяция):
```typescript
const weight = weightA * (1 - morph) + weightB * morph
```

**Строка 57:** Если вес ниже `density` → шаг активен:
```typescript
const active = weight < density
```

Логика: При `density = 1.0` почти все шаги будут активны (так как веса $W \in [0, 1]$).

---

##### b) Выбор ноты (Строки 59-67)

Если шаг активен, выбираем ноту из `scaleNotes`:

$$
noteIdx = \lfloor NoteWeight_i \times |ScaleNotes| \rfloor
$$

**Построчный разбор:**

**Строки 62-64:** Получаем вес для выбора ноты (используем offset `+16` для независимости):
```typescript
const nwA = getWeight(i + 16, seedA)
const noteWeight = nwA * (1 - morph) + nwB * morph
```

**Строка 65:** Индекс ноты:
```typescript
const noteIdx = Math.floor(noteWeight * scaleNotes.length)
```

**Строка 66:** Выбираем ноту из лада:
```typescript
noteName = scaleNotes[noteIdx] || root || 'C'
```

---

##### c) Слайд (Glide/Portamento) (Строки 69-72)

$$
slide_i = active_i \land (SlideWeight_i < T \times 0.5)
$$

**Построчный разбор:**

**Строки 69-71:** Получаем вес для слайда (offset `+32`):
```typescript
const slideWeight = swA * (1 - morph) + swB * morph
```

**Строка 72:** Включаем слайд, если вес мал и параметр `type` высокий:
```typescript
const slide = active && slideWeight < (type * 0.5)
```

Логика: При `type = 1.0` вероятность слайда 50%, при `type = 0.3` — 15%.

---

##### d) Акцент (Accent) (Строки 74-78)

$$
accent_i = active_i \land (AccentWeight_i < Threshold_i)
$$

$$
Threshold_i = 0.3 + OffBeatBias_i
$$

$$
OffBeatBias_i = \begin{cases}
0.2 & \text{if } i \mod 2 = 1 \\
0 & \text{otherwise}
\end{cases}
$$

**Построчный разбор:**

**Строки 74-76:** Генерируем вес акцента (offset `+48`):
```typescript
const accentWeight = awA * (1 - morph) + awB * morph
```

**Строка 77:** Повышенная вероятность акцента на слабых долях (off-beat):
```typescript
const offBeatBias = (i % 2 === 1) ? 0.2 : 0
```

**Строка 78:** Расчёт финального акцента:
```typescript
const accent = active && accentWeight < (0.3 + offBeatBias)
```

Пример: На шаге 1 (слабая доля) порог = 0.5, на шаге 0 (сильная) = 0.3. Это создаёт **грув** (акценты на off-beat характерны для техно).

---

##### e) Формирование шага (Строки 80-86)

**Строка 81:** Добавляем октаву к имени ноты:
```typescript
note: `${noteName}${octave}`
```

Пример: `"C"` + `2` → `"C2"`.

**Строка 82:** Если акцент → громкость 1.0, иначе 0.7:
```typescript
velocity: accent ? 1.0 : 0.7
```

**Строки 83-85:** Сохраняем флаги:
```typescript
slide,
accent,
active
```
