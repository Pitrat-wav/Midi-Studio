# 📘 АУДИТ: DroneEngine.ts

## 1. Глобальное Описание

### Назначение
Этот файл реализует **Drone Engine** (движок текстур) — специализированный синтезатор для создания бесконечных, эволюционирующих атмосферных звуков. Он не использует традиционные мелодии, а полагается на глубокую модуляцию, теорию хаоса и вероятностные алгоритмы (Ворота Бернулли), чтобы создавать "живой" гул, который никогда не повторяется одинаково.

### Связи
- **Импорты:** `tone`.
- **Используется в:** `src/store/audioStore.ts` и управляется через экран Drone.
- **Философия:** Звуковой дизайн через математические константы и физическое моделирование хаоса.

---

## 2. Интерфейс и UI Элементы

Дрон-движок управляется абстрактными параметрами, которые меняют целые группы настроек:
- **Intensity:** Глобальная мощь (громкость, насыщенность эффектов).
- **Chaos:** Управляет частотой случайных изменений в звуке.
- **Nervousness:** Скорость микро-детонации (дрожания) нот.
- **Grit:** Насыщенность шумом и резонансом фильтра.
- **FM Depth:** Степень "металлического" окраса звука.

## 3. Логика и Математика

### 3.1 Слои синтеза и Тритоны

**Строки:** 40-51

Двигатель использует три осциллятора, настроенных по интервалам, создающим напряжение (тритон):

1. **Osc 1:** Базовая частота $f$ (пила).
2. **Osc 2:** $f \times 1.414$ (квадрат). Множитель $\sqrt{2}$ создает математически чистый тритон (диссонанс, характерный для эмбиента).
3. **Sub Osc:** $f \times 0.5$ (синус) для глубокого баса.

---

### 3.2 Хаотическая FM-модуляция

**Строки:** 45-51

Для создания "грязного" нелинейного звука используется FM-модуляция с иррациональным числом Эйлера ($e$):
$$Freq_{Modulator} = f \times 2.718$$

Использование иррациональных множителей предотвращает образование гармонических повторений, делая звук более "аналоговым" и сложным.

---

### 3.3 Механизмы Хаоса (Bernoulli Gates и S&H)

**Строки:** 89-107

Движок содержит внутренний цикл (`Tone.Loop`), который каждые 2 доли (2n) производит случайные действия.

**Математическая модель (Ворота Бернулли):**
Проверка условия с вероятностью $p$:
$$Action = \begin{cases} 
Trigger & \text{если } \text{random} < p \\
Skip & \text{иначе} 
\end{cases}$$

**Примеры в коде:**
- **Sample & Hold (S&H):** С вероятностью 30% выбирается новое случайное значение для фильтра.
- **Glitch:** С вероятностью 10% включается максимальный дисторшн на 0.1 секунды, создавая эффект "битого" звука.

---

### 3.4 Сложная модуляция (LFO)

**Строки:** 138-147

Параметр **Nervousness** управляет частотой LFO, который расстраивает (detune) осцилляторы:
$$LFO_{freq} = 0.05 + Nervousness \times 2$$
$$LFO_{amplitude} = 5 + Nervousness \times 45$$

При максимальном "нервяке" звук начинает сильно вибрировать, уходя от основной частоты на 45-50 центов.
