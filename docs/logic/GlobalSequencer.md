# 📘 АУДИТ: GlobalSequencer.ts

## 1. Глобальное Описание

### Назначение
Этот файл является "сердцем" всей системы. Он реализует глобальный цикл воспроизведения (Sequencer Loop), который синхронизирует все инструменты по времени. Секвенсор работает с точностью до 1/16 доли и отвечает за запуск паттернов, обработку аранжировки и выполнение автоматизаций.

### Связи
- **Импорты:** Собирает состояния из всех магазинов (`audioStore`, `instrumentStore`, `arrangementStore`, `visualStore`).
- **Используемые алгоритмы:** `PadGenerator` для гармоний.
- **Главные функции:** `startSequencerLoop` и `stopSequencerLoop`.

---

## 2. Интерфейс и UI Элементы

❌ **Файл не содержит UI-компонентов**, но он управляет состоянием, которое отображается в интерфейсе:
- **Шаг воспроизведения:** Через `Tone.Draw.schedule` обновляет `currentStep`, что заставляет мигать светодиоды на падах секвенсора.
- **Логи воспроизведения:** Выводит информацию в консоль об текущем такте и запущенных клипах.

## 3. Логика и Математика

### 3.1 Глобальный цикл (startSequencerLoop)

**Строки:** 14-244

**Что делает:**  
Создает объект `Tone.Loop` с интервалом "16n" (шестнадцатая доля). Внутри каждой итерации происходит проверка всех активных инструментов и их паттернов.

**Логические блоки:**
1. **Счетчик шагов:** Использует `stepCounter` для определения позиции внутри 16-шагового паттерна и общего времени композиции.
2. **Логика аранжировки:** Если пользователь находится в режиме "ARRANGE", секвенсор проверяет список клипов и переключает пресеты (snapshots) инструментов в нужный момент.
3. **Триггеры инструментов:** 
   - **Drums:** Проверка сетки 16x16.
   - **Bass:** Отработка паттернов "Sting" с учетом слайдов (slide).
   - **Lead:** Отработка системы "Stages" (шаговый секвенсор с длительностью пульсов).

---

### 3.2 Функция интерполяции автоматизации

**Строки:** 63-77

**Что делает:**  
Позволяет плавно изменять параметры (например, громкость) между двумя точками на временной шкале аранжировки.

**Математическая модель (Линейная интерполяция):**

Если текущий шаг $t$ находится между точками $(t_1, v_1)$ и $(t_2, v_2)$, то значение $v$ рассчитывается как:

$$ratio = \frac{t - t_1}{t_2 - t_1}$$
$$v = v_1 + (v_2 - v_1) \times ratio$$

Где:
- $t$: текущий тик (totalStep).
- $v$: расчетное значение параметра.

---

### 3.3 Преобразование Gain в Decibels

**Строки:** 95, 129, 155, 196, 221

**Что делает:**  
Конвертирует линейное значение громкости (0.0 - 1.0) в логарифмическую шкалу децибел, которую понимает Tone.js.

**Математическая модель:**
$$dB = 20 \times \log_{10}(gain)$$

В коде используется встроенная функция `Tone.gainToDb(autoVol)`.

### 3.4 Построчный разбор цикла

**Строки 22-25:** Извлечение текущего шага (`step` от 0 до 15) и общего времени (`totalStep`).
**Строки 38-49:** Поиск клипов аранжировки, которые должны начаться на текущем шаге. Если найдены — вызывается `triggerSnapshot`.
**Строки 80-85:** Вычисление активности трека с учетом режимов **Solo** и **Mute**. Если хотя бы один трек в Solo, все остальные (не Solo) затихают.
**Строки 172-181:** Специальная логика для Lead секвенсора. Он переходит к следующей ноте не каждый шаг, а через количество шагов, определенное в `stage.pulseCount`.
