# 📘 АУДИТ: DeviceManager.ts

## 1. Глобальное Описание

### Назначение
Этот файл отвечает за управление "Устройствами" (Devices) в модульном редакторе узлов (Node Editor). Документ определяет библиотеку готовых пресетов — заранее собранных схем из осцилляторов, фильтров и эффектов. Также он предоставляет механизмы для сохранения созданных пользователем схем в локальное хранилище браузера (`localStorage`) и их последующей загрузки.

### Связи
- **Store:** Взаимодействует с `nodeStore` для получения данных о текущей схеме.
- **Использование:** Активируется в меню пресетов редактора узлов.
- **Данные:** Использует формат JSON для кодирования графа (узлы + связи).

## 2. Интерфейс и UI Элементы

Модуль обеспечивает работу следующих элементов HUD:
- **Preset Browser:** Список готовых инструментов (например, "Basic Saw Poly" или "Simple Delay").
- **Кнопка "Save Device":** Позволяет сохранить текущую сборку узлов под своим именем.

## 3. Логика и Математика

### 3.1 Определение пресетов (DevicePreset)

**Строки:** 10-46

Каждый пресет — это описание целого подграфа. В нем жестко заданы:
- **Типы узлов:** (например, `audio_osc`, `audio_env`).
- **Координаты (position):** Позиции на холсте для визуального порядка.
- **Параметры (params):** Начальные настройки (частота, атака и др.).
- **Связи (edges):** Логика соединения входов и выходов.

---

### 3.2 Метод `loadPreset(id)`

**Строки:** 49-55

Метод ищет пресет в массиве `PRESETS` по его идентификатору. В случае успеха он передает строку данных в `nodeStore.loadDevice()`. Это вызывает цепную реакцию в `GraphEngine`, который удаляет старые аудио-узлы и создает новые согласно схеме.

---

### 3.3 Сохранение данных (`saveToLocal`)

**Строки:** 57-61

Использует механизмы сериализации (преобразование живых объектов в текст).
**Логика:**
1. Берет текущее состояние графа из `nodeStore`.
2. Сохраняет его в `localStorage` с префиксом `device_`. Это позволяет сохранять настройки даже при перезагрузке страницы.

**Математическая модель данных:**
Граф представляется как ориентированный мультиграф:
$$G = (V, E)$$
Где $V$ (вершины) — это узлы с параметрами, а $E$ (ребра) — это связи между конкретными портами (Handles).
