## 3. ЛОГИКА И МАТЕМАТИКА

### 3.1 Функция: `bjorklund(steps: number, pulses: number): number[]`

**Строки:** 5-50

**Что делает:**  
Генерирует массив из `steps` элементов, где `1` означает удар (pulse), а `0` — паузу (silence). Алгоритм распределяет `pulses` ударов максимально равномерно по `steps` шагам.

**Математическая модель:**

Задача: Распределить $k$ ударов на $n$ шагов:

$$
E(k, n) = \text{Euclidean Pattern}
$$

**Алгоритм Бьоркланда** (рекурсивное деление остатка):

1. Инициализация:
   $$divisor = n - k$$
   $$remainder_0 = k$$

2. Рекурсия:
   $$count_i = \lfloor \frac{divisor}{remainder_i} \rfloor$$
   $$remainder_{i+1} = divisor \mod remainder_i$$
   
3. Построение паттерна методом "группировки":
   - Группы `[0]` и `[1]` комбинируются согласно счётчикам.

**Пример:** $E(5, 13)$  
Результат: `[1, 0, 0, 1, 0, 1, 0, 0, 1, 0, 1, 0, 0]`

**Построчный разбор:**

**Строки 6-7:** Граничные случаи.
- Если ударов 0 или меньше → возвращаем массив пауз.
- Если ударов больше или равно шагам → возвращаем массив сплошных ударов.

**Строки 9-15:** Инициализац

ия переменных.
- `remainders[0] = pulses` (остаток = количество ударов).
- `divisor = steps - pulses` (делитель = количество пауз).

**Строки 17-23:** Цикл вычисления счётчиков (алгоритм Евклида).
- Вычисляем, сколько раз делитель умещается в остаток.
- Сохраняем новый остаток.
- Останавливаемся, когда остаток ≤ 1.

**Строки 27-36:** Рекурсивная функция `buildPattern`.
- **level = -1**: Добавить паузу (`0`).
- **level = -2**: Добавить удар (`1`).
- **Иначе**: Комбинировать группы согласно счётчикам.

**Строки 42-46:** Нормализация паттерна.
- Разворачиваем массив (реверс).
- Ротируем так, чтобы первый удар был в начале (если $k > 0$).

**Строка 49:** Обрезаем массив до `steps` элементов (защита от переполнения).

### 3.2 Функция: `rotateArray<T>(arr: T[], steps: number): T[]`

**Строки:** 55-60

**Что делает:**  
Циклически сдвигает массив на `steps` позиций.

**Математическая модель:**

$$
rotate(arr, s) = arr[(n - offset):] + arr[:(n - offset)]
$$

Где:
$$
offset = ((s \mod n) + n) \mod n
$$

Двойное взятие остатка гарантирует корректную обработку отрицательных сдвигов.

**Построчный разбор:**

**Строка 56:** Проверка пустого массива.

**Строка 58:** Вычисление безопасного сдвига:
- `steps % n` → нормализуем большие значения.
- `+ n` → обрабатываем отрицательные числа.
- `% n` → финальная нормализация.

**Строка 59:** Конкатенация двух частей массива:
- Берём хвост: `arr.slice(n - offset)`.
- Добавляем голову: `arr.slice(0, n - offset)`.
