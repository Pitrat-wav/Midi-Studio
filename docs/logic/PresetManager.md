# 📘 АУДИТ: PresetManager.ts

## 1. Глобальное Описание

### Назначение
Этот файл реализует систему **Снапшотов** (мгновенных снимков состояния) всей студии. **PresetManager** позволяет сохранить абсолютно все параметры — от темпа и тональности до положения каждой ручки на модульном синтезаторе — в один объект данных (JSON). Это ядро системы сохранения проектов и автоматизации в режиме аранжировки.

### Связи
- **Импорты:** Собирает данные изо всех магазинов (`audioStore`, `drumStore`, `bassStore`, `sequencerStore`, `harmStore`, `padStore`, `harmonyStore`).
- **Используется в:** Секции снапшотов HUD и в `GlobalSequencer.ts` для автоматического переключения звуковых состояний во время проигрывания трека.

---

## 2. Интерфейс и UI Элементы

Этот модуль обслуживает следующие функции интерфейса:
- **Кнопки Snapshots (1-8):** Позволяют мгновенно переключаться между разными звуковыми сценами.
- **Arrangement Clips:** Каждый блок на временной шкале аранжировщика содержит ID снапшота, который `PresetManager` применяет в нужный момент.

## 3. Логика и Математика

### 3.1 Структура снапшота (GlobalSnapshot)

**Строки:** 11-61

Снапшот представляет собой глубокую иерархию данных. Важной особенностью является **сериализация**: перед сохранением из объектов удаляются все функции и ссылки на живые аудио-узлы, оставляя только чистые числа и строки.

---

### 3.2 Метод `capture()` (Захват состояния)

**Строки:** 64-127

Этот метод "опрашивает" все магазины Zustand и собирает их текущие значения.

**Особенности реализации:**
- Используется деструктуризация (строка 74) для исключения методов (`setParam`, `loadPreset` и др.) из сохраняемых данных.
- Для глубокого копирования сложных объектов используется хак: `JSON.parse(JSON.stringify(data))`. Это гарантирует, что изменения в текущем состоянии не затронут уже сохраненный снимок.

---

### 3.3 Метод `apply(snapshot)` (Применение состояния)

**Строки:** 129-190

Метод восстанавливает состояние студии из сохраненного снимка. Он последовательно вызывает методы `setState` или специализированные сеттеры во всех магазинах.

**Математическая логика восстановления:**
Применение происходит поэтапно, чтобы избежать конфликтов:
1. **Тайминг:** Сначала BPM и свинг.
2. **Микс:** Громкости и Mute-статусы.
3. **Генераторы:** Паттерны барабанов и баса.
4. **Синтез:** Параметры модульного движка.

Это гарантирует, что аудио-движок плавно перейдет к новым настройкам (через методы `rampTo`, если они используются в сеттерах).
