# withHistory.ts

### 1. Глобальное Описание (File Overview)
* **Назначение:** Middleware для Zustand, перехватывающий все изменения состояния (State updates). Он вычисляет разницу (diff) между старым и новым состоянием и отправляет эти изменения в глобальный `historyStore`.
* **Связи:** 
    * Импортирует: `useHistoryStore`, `registerStore` из `../historyStore`.
    * Используется в: `instrumentStore.ts` (оборачивает сторы Drums, Bass, Harm, Sequencer).

### 2. Интерфейс и API
* **`WithHistoryOptions`** (Interface):
    * `storeName`: Уникальный идентификатор стора (строка).
    * `ignorePaths`: Массив путей, изменения в которых игнорируются (например, `['analyzer', 'visual.energy']`).

* **`withHistory`** (Function):
    * High-Order Function, оборачивающая инициализатор Zustand стора.
    * Переопределяет функцию `set`, добавляя в неё логику сравнения состояний.

### 3. Логика и Математика (Logic & Math)

* **Функция:** `getDiff(obj1, obj2, basePath, ignorePaths)`
* **Что делает:** Рекурсивно сравнивает два объекта и находит изменившиеся конечные значения.
* **Алгоритм:**
    1. Получает объединение всех ключей из `obj1` и `obj2`.
    2. Для каждого ключа проверяет, не входит ли путь в `ignorePaths`.
    3. Если значения равны (`===`), пропускает.
    4. Если оба значения — объекты (и не массивы), вызывает `getDiff` рекурсивно.
    5. Иначе выполняет "грязную" проверку через `JSON.stringify`. Если строки отличаются, фиксирует изменение.

* **Математическая модель (Diff Logic):**
    Пусть $S_{old}$ — старое состояние, $S_{new}$ — новое состояние.
    Множество изменений $\Delta = \{ (path, val_{old}, val_{new}) \}$, где:
    $$ \Delta = \bigcup_{k \in Keys} \begin{cases} \emptyset, & \text{if } S_{old}[k] = S_{new}[k] \lor k \in Ignore \\ getDiff(S_{old}[k], S_{new}[k]), & \text{if isObject}(S_{old}[k]) \\ \{ (k, S_{old}[k], S_{new}[k]) \}, & \text{otherwise} \end{cases} $$

* **Построчный разбор (`newSet`):**
    1. **Захват:** Сохраняем `oldState = get()`.
    2. **Выполнение:** Вызываем оригинальный `set(...args)`.
    3. **Захват:** Сохраняем `newState = get()`.
    4. **Сравнение:** Если ссылки равны, выход. Иначе вызываем `getDiff`.
    5. **Отправка:** Если есть изменения:
        * Если 1 изменение -> `historyStore.pushAction`
        * Если >1 изменений -> Оборачиваем в транзакцию (`startTransaction` -> `push` -> `endTransaction`).
