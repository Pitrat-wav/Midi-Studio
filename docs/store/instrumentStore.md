# 📘 АУДИТ: instrumentStore.ts

## 1. Глобальное Описание

### Назначение
Этот файл содержит коллекцию специализированных хранилищ (stores) для каждого типа музыкального инструмента в приложении. Он определяет "память" для барабанов, баса, секвенсоров и модульных синтезаторов. Здесь хранятся все параметры синтеза, паттерны секвенсоров и музыкальные настройки (тональность, лад).

### Связи
- **Импорты:** Использует логику из `bjorklund`, `GridWalker`, `StingGenerator`, `Scaler`.
- **Используется в:** Всех UI-экранах инструментов и в `GlobalSequencer` для получения текущих данных для воспроизведения.

---

## 2. Интерфейс и UI Элементы

### 2.1 Магазин Harmony (useHarmonyStore)
Управляет глобальным музыкальным контекстом:
- **Root:** Корневая нота (C, D, E...).
- **Scale:** Музыкальный лад (Minor, Major, Dorian...).

### 2.2 Магазин Drums (useDrumStore)
Управляет эвклидовым драм-секвенсором:
- **Steps/Pulses:** Количество шагов и ударов для каждого барабана (Kick, Snare, HiHat и т.д.).
- **Kit:** Переключение между эмуляциями 808 и 909.

### 2.3 Магазин Bass (useBassStore)
Управляет параметрами "кислотного" баса:
- **Density/Type:** Параметры генерации паттерна Sting.
- **Cutoff/Resonance:** Регуляторы фильтра.

### 2.4 Магазин Sequencer (useSequencerStore)
Комбинированный магазин для трех типов секвенсоров:
- **Stages:** Индивидуальные настройки шагов (Pitch, Gate Mode, Repeat).
- **Snake:** Сетка 4x4 с выбором паттерна обхода.
- **Turing:** Вероятность изменения мелодии (Probability) и фиксация цикла (Lock).

### 2.5 Магазин Harm (useHarmStore)
Магазин для сложного модульного синтезатора. Содержит **52 готовых пресета** в разных категориях (Bass, Lead, Pad, SFX).

## 3. Логика и Математика

### 3.1 Динамический расчет паттернов (Drums)

**Строки:** 70-78

Каждый раз, когда пользователь меняет количество ударов (`pulses`) или шагов (`steps`) в UI, магазин автоматически пересчитывает эвклидов паттерн:
```typescript
if (params.pulses !== undefined || params.steps !== undefined) {
    const pattern = bjorklund(newState[drum].steps, newState[drum].pulses)
    newState.activePatterns[drum] = pattern.map(v => v === 1)
}
```

---

### 3.2 Система морфинга семян (Bass)

**Строки:** 156-175

Использует два "семени" случайности (`seedA` и `seedB`) для создания бесконечного разнообразия мелодий. Параметр `morph` плавно интерполирует между ними:
$$ResultWeight = Weight(seedA) \times (1 - morph) + Weight(seedB) \times morph$$

---

### 3.3 Структура Stages (Ступенчатый секвенсор)

**Строки:** 186-194

Каждый шаг (`Stage`) имеет сложную структуру:
- **pitch:** Высота звука.
- **pulseCount:** Количество повторений ноты на одном шаге секвенсора.
- **gateMode:** Режим удержания ноты (Mute, Single, Multi, Hold).

---

### 3.4 Пресеты Harm Synth

**Строки:** 434-579

Этот раздел содержит огромный массив конфигураций для 3-х осцилляторного синтезатора с шумом и 2-мя фильтрами. Каждый пресет определяет:
- Типы волн для каждого осциллятора.
- Настройки огибающих ADSR.
- Настройки эффектов (Chorus, Phaser, Distortion).
- Параметры West Coast синтеза (Buchla 259): FM Index, Timbre, Fold.
