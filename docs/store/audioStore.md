# 📘 АУДИТ: audioStore.ts

## 1. Глобальное Описание

### Назначение
Этот файл является центральным диспетчерским узлом аудио-системы. Он использует **Zustand** для управления всем состоянием звука в приложении. Здесь происходит инициализация аудио-движка Tone.js, создание всех виртуальных инструментов, настройка микрофона, управление эффектами (реверберация, дилэй, эквалайзер) и динамическая маршрутизация сигналов.

### Связи
- **Импорты:** Собирает все классы синтезаторов из `src/logic/` (`AcidSynth`, `DrumMachine`, `PadSynth`, `FMBass`, `HarmSynth`, `SamplerInstrument`).
- **Зависимости:** `tone` (аудио-движок), `zustand` (стейт-менеджмент).
- **Связанные хранилища:** Взаимодействует с другими магазинами (`useBassStore`, `useDrumStore` и др.) для синхронизации параметров.

## 2. Интерфейс и UI Элементы

Этот магазин данных питает 90% пользовательского интерфейса:

- **Глобальный транспорт:** Ручки **BPM** и **Swing**.
- **Микшерный пульт:** Слайдеры громкости (`volumes`), кнопки **Mute** и **Solo** для каждого канала (Drums, Bass, Lead, Pads, Harm, Sampler, Mic).
- **Секция эффектов:** Слайдеры для Reverb (Wet/Decay), Delay (Wet/Feedback), Distortion (Wet/Amount).
- **Мастер-эквалайзер:** 4 полосы (Low, Low-Mid, High-Mid, High).
- **Секция микрофона:** Кнопка включения и мониторинга.
- **Индикаторы загрузки:** `loadingStep` отображается на загрузочном экране.

## 3. Логика и Математика

### 3.1 Функция: `initialize()`

**Строки:** 104-305

**Что делает:**  
Запускает крайне сложную цепочку асинхронных событий для подготовки аудио-движка.

**Математическая модель (Signal Chain):**
$$MasterBus \rightarrow EQ_{4Band} \rightarrow Distortion \rightarrow Delay \rightarrow Reverb \rightarrow Logic(Destination)$$

**Построчный разбор:**
1. **Unlocking Audio:** (Строки 112-116) — Специальный хак для iOS. Проигрывает бесшумный WAV-файл в `data:uri`, чтобы "разбудить" аудио-контекст браузера.
2. **Master EQ:** (Строки 129-140) — Создает четыре фильтра (LowShelf, Peaking, HighShelf).
3. **FX Chain:** (Строка 160) — Соединяет все эффекты в последовательную цепь методом `chain()`.
4. **Микрофон:** (Строки 164-170) — Mic -> Gate (отсечка шума) -> Compressor -> Gain.
5. **Синтезаторы:** (Строки 182-211) — Асинхронно создает экземпляры всех инструментов с искусственной задержкой 10мс между ними, чтобы не "повесить" UI-поток.

---

### 3.2 Функция: `recalculateRouting(edges)`

**Строки:** 564-632

**Что делает:**  
Это ядро **Node Editor**. Она перестраивает аудио-граф в реальном времени на основе связей (edges) из визуального редактора.

**Логика:**
1. Отключает все инструменты и эффекты от всего (`disconnect`).
2. Сопоставляет ID узла из визуального интерфейса с реальным объектом Tone.js (`getToneNode`).
3. Для каждого соединения в массиве `edges` выполняет:
   $$sourceNode.connect(targetNode)$$
4. Если соединений нет, восстанавливает стандартную линейную схему.

---

### 3.3 Функция: `triggerPerformanceFx(effect, active)`

**Строки:** 440-478

**Что делает:**  
Запускает макро-команды для живого выступления.

**Математические модели эффектов:**
- **Tape Stop:** Плавное замедление BPM до 10 и уменьшение громкости до нуля за 0.5с.
  $$BPM_{ramp} = 10, \Delta t = 0.5s$$
- **Stutter:** (Строки 456-468) — Быстрое переключение `mute` мастер-выхода через `setInterval(100ms)`.

---

### 3.4 Функции Снапшотов (Snapshots)

**Строки:** 634-675

**Что делает:**  
Управляет мгновенным переключением всех параметров инструментов (целыми наборами настроек).

**Логика:**
1. `triggerSnapshot`: Ставит в очередь индекс пресета.
2. `commitSnapshots`: Вызывается секвенсором в начале каждого такта. Берет данные из библиотеки `SNAPSHOT_LIBRARY` и распределяет их по соответствующим под-магазинам (`useDrumStore`, `useBassStore` и т.д.).

- **Логика:** Метод `syncAudioClips` итерирует по всем аудио-клипам и вызывает `player.sync().start(pos)`, где `pos` рассчитывается на основе тиков клипа. Это позволяет аудио "прилипать" к сетке Transport.
- **Математика:** Расчет времени старта в секундах на основе тиков:
  $$StartSeconds = StartTick \times \left(\frac{60}{BPM \times 4}\right)$$
  (Где 4 — это количество 16-х долей в четверти).
