# themeStore.ts

### 1. Глобальное Описание (File Overview)
* **Назначение:** Глобальное управление цветовыми схемами (темами) приложения. Позволяет переключаться между 50+ пресетами (Cyberpunk, Matrix, Nord и др.) или генерировать новые темы процедурно.
* **Связи:** 
    * Управляет: CSS-переменными в `document.documentElement`.
    * Влияет на: Весь UI (фоны, тексты, границы, ноды).

### 2. Интерфейс и Параметры (Theme Presets)

* **Реестр: `PRESETS`**
    * **Где находится:** Строки 36-87.
    * **Логика:** Список из более чем 50 предустановленных тем с уникальными цветовыми палитрами.

### 3. Логика и Математика (Logic & Math)

* **Функция: `generateRandomTheme(id)`**
* **Что делает:** Генерирует гармоничную цветовую схему на основе случайного оттенка (Hue).
* **Математическая модель (Color Theory):**
    $$ Hue_{primary} = Random(0 \dots 359) $$
    $$ Hue_{secondary} = (Hue_{primary} + 180) \pmod{360} \quad \text{(Complementary)} $$
    $$ Hue_{accent} = (Hue_{primary} + 90) \pmod{360} \quad \text{(Triadic/Analogous)} $$
    Цвета формируются в формате HSL: `hsl(H, S%, L%)`.

* **Функция: `applyTheme(theme)`**
* **Что делает:** Переносит значения цветов из JS-объекта в CSS-переменные.
* **Построчный разбор:**
    1. Находим корневой элемент `documentElement`.
    2. Вызываем `style.setProperty` для каждой переменной:
        * `--bg-color`
        * `--primary-color`
        * `--node-bg` (мапинг для Node Editor)
        * и т.д.
