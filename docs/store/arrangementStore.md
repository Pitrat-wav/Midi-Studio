# arrangementStore.ts

### 1. Глобальное Описание (File Overview)
* **Назначение:** Хранилище структуры музыкальной композиции (Timeline). Управляет клипами (фрагментами), маркерами и автоматизацией параметров на временной шкале.
* **Связи:** 
    * Импортирует: `zustand`.
    * Используется в: `ArrangementEditor.tsx` и Audio-движке для воспроизведения структуры.

### 2. Интерфейс и Параметры (Arrangement Elements)

* **Сущность: Клип (`ArrangementClip`)**
    * **Свойства:** `startTick` (позиция), `durationTicks` (длительность), `snapshotId` (состояние инструментов).

* **Сущность: Маркер (`SongMarker`)**
    * **Свойства:** Метка времени и название (INTRO, DROP и т.д.).

* **Сущность: Автоматизация (`AutomationPoint`)**
    * **Логика:** Значение параметра в конкретный момент времени (`tick`).

### 3. Логика и Математика (Logic & Math)

* **Функция:** `splitClip(id, tick)`
* **Что делает:** Разрезает клип на две части в указанной точке.
* **Математическая модель:**
    $$ Clip_1.Duration = Tick - Clip.Start $$
    $$ Clip_2.Start = Tick $$
    $$ Clip_2.Duration = (Clip.Start + Clip.Duration) - Tick $$

* **Функция:** `moveSelectedClips(deltaTicks, deltaTrack)`
* **Построчный разбор:**
    1. Находит все выбранные клипы по `selectedClipIds`.
    2. Применяет смещение по времени: `Math.max(0, startTick + deltaTicks)`.
    3. Применяет смещение по дорожкам: находит индекс текущего трека в `TRACK_ORDER` и прибавляет `deltaTrack`.

* **Математика Времени (Ticks):**
    Приложение использует сетку в 16-е доли.
    $$ 1 \text{ Bar} = 16 \text{ Ticks} $$
    $$ zoomLevel = \text{Ticks Per Pixel} $$

### Глубокий разбор логики и математики
- **Логика:** Теперь треки могут быть либо инструментальными (MIDI), либо аудио-треками. Это разделение позволяет `AudioTrackManager` фильтровать клипы для создания плееров.
- **Математика:** Поле `peaks` представляет собой нормализованный массив амплитуд (100 точек) для мгновенного рендеринга волновой формы без повторного анализа файла.
