# 📘 АУДИТ: visualStore.ts

## 1. Глобальное Описание

### Назначение
Этот файл является командным центром для всей визуальной части приложения. Он управляет состоянием 3D-сцены, освещением, эффектами окружения (туман, звезды), режимами работы камеры и данными от системы отслеживания рук (Hand Tracking). Store также отвечает за "визуальную энергию" — данные, которые заставляют 3D объекты пульсировать в такт музыке.

### Связи
- **Импорты:** Использует `zustand` для хранения состояния.
- **Используется в:** Практически во всех 3D компонентах (`src/components/WebGL/...`) для получения данных об интенсивности звука и текущем фокусе камеры.
- **Взаимодействие:** Обновляется через `AudioVisualBridge` и `useHandTracking`.

## 3. Логика и Математика

### 3.1 Система "Энергии" и "Пульса"

**Строки:** 37-45, 91-102

**Что делает:**  
Хранит мгновенные значения интенсивности для каждого инструмента. Когда в аудио-движке происходит удар (например, бочка), значение `triggers.kick` устанавливается в 1.0, а затем плавно затухает.

---

### 3.2 Функция затухания: `decay()`

**Строки:** 202-230

**Что делает:**  
Эта функция вызывается каждый кадр (в цикле `requestAnimationFrame`) для плавного уменьшения значений яркости и масштаба объектов после того, как они были "триггернуты" звуком.

**Математическая модель (Linear Decay):**
Для каждого визуального параметра $V$:

$$V_{next} = \max(0, V_{current} - R)$$

Где $R$ — скорость затухания (`rate`).

**Построчный разбор:**
1. Определяет вспомогательную функцию `d(val, rate)`.
2. Обновляет объект `energy` (медленное затухание 0.05).
3. Обновляет объект `triggers` (быстрое затухание 0.15 - 0.2 для резких вспышек).
4. Уменьшает `globalAudioIntensity` для затухания общих эффектов сцены.

---

### 3.3 Функция переключения видов: `cycleView()`

**Строки:** 160-165

**Что делает:**  
Циклически меняет текущий экран приложения.

**Математическая модель:**
$$Index_{next} = (Index_{current} + 1) \mod TotalViews$$

---

### 3.4 Информационные структуры

- **fftData (Float32Array):** Сырые данные частотного анализа звука для отрисовки аудио-спектра.
- **handData (HandLandmark[]):** Массив координат $x, y, z$ для 21 точки каждой руки, полученных от нейросети MediaPipe.
