# usePresetStore.ts

### 1. Глобальное Описание (File Overview)
* **Назначение:** Хранилище пользовательских пресетов (снимков состояния системы). Позволяет сохранять текущие настройки всех инструментов и визуализаторов в именованный пресет и загружать их позже.
* **Связи:** 
    * Импортирует: `GlobalSnapshot`, `PresetManager`.
    * Взаимодействует с: `useVisualStore` для отображения уведомлений.

### 2. Интерфейс и API (Core API)

* **Метод: `saveCurrentAs(name)`**
    * **Логика:** Делает полный снимок состояния через `PresetManager.capture()` и сохраняет его с новым ID.

* **Метод: `loadPreset(id)`**
    * **Логика:** Находит данные пресета по ID и применяет их к системе через `PresetManager.apply()`.

### 3. Логика и Математика (Logic & Math)

* **Функция: Создание ID**
    * **Логика:** Генерирует уникальную строку случайных символов.
    * **Математика:**
        $$ ID = RandomString(base36).substring(2, 9) $$

* **Построчный разбор (`saveCurrentAs`):**
    1. Вызываем `PresetManager.capture()` — это ресурсозатратная операция, собирающая данные из всех Zustand-сторов.
    2. Создаем объект `newPreset`.
    3. Обновляем массив `presets` в сторе.
    4. Вызываем `setStatus` в `visualStore` для визуального подтверждения ("PRESET SAVED").
