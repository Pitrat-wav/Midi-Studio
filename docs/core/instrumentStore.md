# 📘 АУДИТ: instrumentStore.ts

## 1. Глобальное Описание

### Назначение
`instrumentStore.ts` — это "банк памяти" Studio Elite 5.0. В этом файле сосредоточены все сторы (Zustand), отвечающие за состояние конкретных инструментов: Drums, Bass, Pads, Harmony Station, Sequencers и Sampler. Здесь хранятся паттерны, настройки синтеза, выбранные шкалы (Scales) и пресеты. Стор обеспечивает реактивность интерфейса и синхронизацию 2D HUD с 3D WebGL объектами.

---

## 2. Ключевые Сторы и Подсистемы

### Harmony Store (Глобальная гармония)
**Строки: 12-28**
Задает тональность (`root`) и лад (`scale`) для всего приложения. Все мелодические инструменты (Bass, Lead, Harmony) обращаются к этому стору для квантования нот.

### Drum Store (Евклидова ритмика)
**Строки: 31-92**
Реализует логику классических драм-машин (808/909).
- **Алгоритм Бьёрклунда:** Используется функция `bjorklund(steps, pulses)` для автоматической генерации "математически идеальных" евклидовых ритмов на лету.

### Bass Store (Acid/FM движки)
**Строки: 112-183**
Хранит параметры для двух типов синтеза: Acid (303-style) и FM (Metallic). Содержит паттерны `BassStep`, генерируемые внешними алгоритмами.

### Sequencer Store (ML-185 + Snake + Turing)
**Строки: 185-328**
Самый сложный стор секвенсоров, объединяющий:
- **ML-185:** 8-шаговый секвенсор с микро-повторами (`pulseCount`) и условиями срабатывания (`condition`).
- **MDD Snake:** Сетка 4x4 с 2D-навигацией.
- **Turing Machine:** Вероятностный сдвиговый регистр для генерации бесконечных вариаций мелодий.
- **Smart Chord:** Логика автоматического построения аккордов по заданному типу (`triad`, `7th`).

### Harm Store (Buchla-Modular)
**Строки: 344-614**
Стор для самого мощного синтезатора. Включает:
- **52 встроенных пресета:** Категории Bass, Pad, Lead, SFX, West Coast.
- **West Coast Synthesis:** Параметры Wavefolding (Timbre, Order, Harmonics) и FM/AM модуляции.
- **Global FX Rack:** Настройки дисторшна, фазера, хоруса, дилея и реверберации внутри инструмента.

### Sampler Store (Granular Engine)
**Строки: 634-707**
Управляет библиотекой семплов (Sample Manifest) и гранулярным движком.

## 3. Математика и Специфическая Логика

### 3.1 Евклидовы ритмы (Bjorklund)

**Строки: 72-76**

При каждом изменении количества шагов (`steps`) или импульсов (`pulses`), стор мгновенно пересчитывает ритмическую сетку:
$$ActivePattern = \text{bjorklund}(Steps, Pulses)$$
Это позволяет создавать сложные полиритмические структуры одним движением слайдера.

### 3.2 Машина Тьюринга (Shift Register)

**Строки: 316-320**

Регистр представлен как 16-битное число. Математика мутации битов на каждом шаге (в логике `audioEngine`):
$$Register = (Register \gg 1) | ((\text{random} < Probability ? \text{newBit} : \text{lastBit}) \ll 15)$$
Стор хранит только текущее состояние этого регистра для визуализации в `Sequencer3D`.

### 3.3 Генерация пресетов (Variation Logic)

**Строки: 538-579**

Используется функциональное программирование для генерации десятков вариаций пресетов на основе базового шаблона (`BASE_PRESET`), программно меняя типы осцилляторов, частоты фильтров и посылы на эффекты.
