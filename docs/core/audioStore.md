# 📘 АУДИТ: audioStore.ts

## 1. Глобальное Описание

### Назначение
`audioStore.ts` — это центральное ядро управления звуком в приложении. Реализованный с помощью `Zustand`, этот стор инкапсулирует всю сложность взаимодействия с библиотекой `Tone.js`. Он отвечает за инициализацию аудио-контекста, создание и связывание инструментов, управление эффектами, микширование каналов, работу с микрофоном и систему квантованного переключения снепшотов. Это "единая точка истины" для любого аудио-события в системе.

### Ключевые Подсистемы
- **Initialize Engine:** Пошаговая загрузка модулей с обратной связью для UI (`loadingStep`).
- **Global FX Chain:** Продвинутая цепь обработки (4-Band EQ, Distortion, Delay, Reverb).
- **Mixer Logic:** Управление громкостью, Mute и Solo для 7 независимых каналов.
- **Microphone Chain:** Интеграция живого входа с шумодавами (Gate) и компрессией.
- **Node Routing:** Возможность динамической перекоммутации модулей (для Node Editor).
- **Snapshot Manager:** Система подготовки (`queue`) и фиксации (`commit`) изменений настроек инструментов синхронно с музыкальной сеткой.

## 2. Логика Инициализации

**Строки: 104-305**

Процесс инициализации является критически важным и включает:
1. **Unlock Context:** Обход ограничений iOS/браузеров через воспроизведение "тихого аудио".
2. **Master Bus Construction:** Создание мастер-шины и подключение цепи эффектов.
3. **Instrument Instantiation:** Создание экземпляров `AcidSynth`, `DrumMachine`, `PadSynth` и др.
4. **Volume Normalization:** Применение начальных уровней децибел для предотвращения клиппинга.
5. **Transport Start:** Запуск глобальных часов `Tone.Transport`.

## 3. Математика и Технические Решения

### 3.1 Децибелы и Гейн (Volume Mapping)

**Строки: 372-381**

При изменении громкости в UI (значение $0..1$) происходит логарифмическое преобразование в децибелы для корректного восприятия человеческим ухом:
$$Volume_{dB} = 20 \times \log_{10}(Value_{0..1})$$
Это реализовано через встроенную функцию `Tone.gainToDb`.

### 3.2 Динамическая маршрутизация (Routing)

**Строки: 564-632**

Функция `recalculateRouting` позволяет полностью перестроить аудио-граф на лету. Она отключает все существующие связи (`disconnect`) и заново прокладывает пути на основе массива ребер (`edges`) из `NodeEditor`. Это превращает приложение из фиксированного микшера в модульную среду.

### 3.3 Квантование снепшотов (Snapshot Sync)

**Строки: 634-675**

Для предотвращения резких скачков звука при смене настроек используется двухфазная система:
1. `triggerSnapshot`: Записывает намерение сменить пресет в `queuedSnapshots`.
2. `commitSnapshots`: Вызывается внешним планировщиком (обычно в конце такта) для одновременного применения всех накопленных параметров.

## 4. Глобальные Эффекты (Performance FX)

**Строки: 440-478**

Реализованы сложные алгоритмические макросы:
- **Tape Stop:** Одновременное замедление BPM до 10 и увод громкости в $-\infty$ за 0.5с через `rampTo`.
- **Wash Out:** Экстремальное увеличение Wet-уровня реверберации и дилея для создания "туманных" переходов.
- **Stutter:** Быстрое циклическое переключение флага `mute` мастера (100мс) через `setInterval`.
