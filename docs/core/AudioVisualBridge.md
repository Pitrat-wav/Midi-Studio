# 📘 АУДИТ: AudioVisualBridge.ts

## 1. Глобальное Описание

### Назначение
`AudioVisualBridge.ts` — это центральная нервная система Studio Elite 5.0, обеспечивающая идеальную синхронизацию между аудио-движком (Tone.js) и визуализацией (Three.js/WebGL). Модуль выступает в роли "диспетчера событий", который перехватывает MIDI-сигналы, анализирует частотный спектр и передает эти данные в 3D-компоненты. Без этого моста 3D-сцена оставалась бы статичной, не реагируя на ритм и тембр музыки.

### Ключевые Подсистемы
- **Event Bus:** Шина событий для распространения MIDI-данных (`Note On/Off`, `CC`).
- **Spectral Analyser:** FFT-анализ реального времени для разделения звука на НЧ, СЧ и ВЧ диапазоны.
- **Uniform Manager:** Подготовка данных для передачи в GPU-шейдеры (`uTime`, `uAudioIntensity`).
- **Timing Sync:** Использование `Tone.Draw` для компенсации задержек браузера и обеспечения визуального отклика точно в момент звукового удара.

## 2. Логика Анализа (Update Loop)

**Строки: 122-186**

В каждом графическом кадре (RAF) мост выполняет:
1. **FFT Sampling:** Получение 512 спектральных полос из мастер-выхода.
2. **RMS Calculation:** Вычисление среднеквадратичной громкости (Root Mean Square) для общей интенсивности.
3. **Frequency Splitting:** Разделение спектра на три зоны:
    - **Low:** ~0–2кГц (мощность баса и бочки).
    - **Mid:** ~2–8кГц (голос, синтезаторы).
    - **High:** 8кГц+ (хэты, воздух).
4. **Pulse Decay:** Плавное уменьшение накопленных импульсов для визуальной мягкости.

## 3. Математика и Технические Решения

### 3.1 Расчет громкости (RMS)

**Строки: 132-137**

Для определения реальной "энергии" звука используется сумма квадратов амплитуд выборок волноформы:
$$RMS = \sqrt{\frac{\sum_{i=0}^{N} Sample_i^2}{N}}$$
Это дает более стабильное значение интенсивности, чем просто пиковая громкость.

### 3.2 Синхронизация через Tone.Draw

**Строки: 231-259**

Проблема веба — рассинхронизация звукового потока и графического кадра. Мост решает это через `Tone.Draw`, который планирует визуальные события в соответствии с аудио-часами:
`Tone.Draw.schedule(() => { triggerPulse(...) }, actualTime)`
Это гарантирует, что 3D-объект вспыхнет именно тогда, когда пользователь услышит атаку звука, независимо от загрузки процессора.

### 3.3 Шейдерные Uniforms

**Строки: 159-163**

Данные упаковываются в структуру `UniformData`, которая затем напрямую считывается материалами в 3D-сцене:
- `uTime`: Глобальное время для процедурной анимации.
- `uAudioIntensity`: Глобальный коэффициент масштабирования объектов.
- `uLowFreq`: Коэффициент искажения (distortion) для пульсирующих басовых элементов.
