# 📘 АУДИТ: GlobalSequencer.ts

## 1. Глобальное Описание

### Назначение
`GlobalSequencer.ts` — это главный "часовой механизм" и планировщик событий Studio Elite 5.0. Модуль реализует основной цикл воспроизведения (`Tone.Loop`), который срабатывает каждые 1/16 такта. Он объединяет данные из всех сторов (Drums, Bass, Harmony, Arrangement) и превращает их в команды для синтезаторов. Этот файл является местом, где "код становится музыкой", отвечая за пошаговое исполнение паттернов, применение автоматизаций и синхронизацию всех систем.

### Ключевые Подсистемы
- **Transport Loop:** Цикл `16n`, работающий в реальном времени с компенсацией задержек.
- **Snapshot Committer:** Механизм фиксации изменений параметров точно в начале каждого такта (`step === 0`).
- **Arrangement Engine:** Логика воспроизведения клипов и переключения состояний инструментов на основе таймлайна.
- **Automation Logic:** Линейная интерполяция значений параметров (например, громкости) между кадрами.
- **Instrument Triggers:** Индивидуальная логика для каждого типа синтезатора (Acid, FM, Pads, Harm).

## 2. Логика Секвенирования

**Строки: 22-238**

Основной цикл работает по следующему алгоритму:
1. **Clock Tick:** Увеличение `stepCounter` и расчет текущего шага внутри 16-шаговой сетки.
2. **Snapshot Sync:** Раз в 16 шагов вызывается `commitSnapshots`, чтобы плавно применить новые настройки пользователю.
3. **Arrangement Check:** Если активен режим `ARRANGE`, проверяется наличие клипов на текущем тике и планируется их активация.
4. **Dynamic Volumes:** Расчет интерполированных значений автоматизации и применение Solo/Mute логики.
5. **Sequential Trigger:** Последовательный опрос всех инструментов и вызов их методов `triggerNote` или `triggerDrum`.

## 3. Математика и Технические Решения

### 3.1 Линейная интерполяция автоматизации (Lerp)

**Строки: 63-77**

Для плавного изменения громкости между двумя точками на таймлайне используется функция `getInterpolatedValue`. Если текущий тик ($t$) находится между точками $P_1(t_1, v_1)$ и $P_2(t_2, v_2)$, значение вычисляется как:
$$Ratio = \frac{t - t_1}{t_2 - t_1}$$
$$Value = v_1 + (v_2 - v_1) \times Ratio$$

### 3.2 Логика Solo/Mute

**Строки: 80-85**

Система использует приоритет Solo: если активен хотя бы один Solo-канал, все остальные каналы затыкаются, независимо от их флага Mute. Это реализовано через функцию `isTrackActive`.

### 3.3 Модульная логика "Пульсов" (Stages Pulse)

**Строки: 158-182**

Для секвенсора ML-185 реализована логика вложенных циклов. На одном "шаге" может быть несколько "пульсов" (повторов ноты). Это управляется через счетчик `stagePulse`, который сбрасывается только после того, как нота прозвучала заданное количество раз (`stage.pulseCount`).
