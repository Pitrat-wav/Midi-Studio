# 📘 АУДИТ: GraphEngine.ts

## 1. Глобальное Описание

### Назначение
`GraphEngine.ts` — это "Deep Core DSP" (цифровой сигнальный процессор) приложения. Модуль отвечает за преобразование визуального графа узлов из **Node Editor** в реальную цепочку обработки звука и данных. Он реализует динамическое создание, обновление и коммутацию аудио-модулей, логических операторов и кастомных JavaScript-скриптов. Это самая гибкая часть системы, позволяющая пользователю строить собственные синтезаторы и эффекты на лету.

### Ключевые Подсистемы
- **Node Lifecycle Manager:** Подписка на `nodeStore` для автоматического создания и уничтожения объектов при изменениях в UI.
- **Audio Wrapper System:** Обертка над объектами `Tone.js`, сопоставляющая визуальные "порты" (Handles) с реальными аудио-параметрами (`frequency`, `gain`, `Q`).
- **JIT Scripting (Just-In-Time):** Выполнение кода пользователя в реальном времени через `ScriptProcessorNode` для кастомной математики.
- **Dynamic Rebuilder:** Полная перекоммутация аудио-графа при изменении связей (Edges) между узлами.

## 2. Логика Узлов (Node Mapping)

**Строки: 93-690**

Движок поддерживает несколько категорий узлов:
1. **Audio Sources:** Осцилляторы, LFO, Драм-модули (Kick/Snare/Hat).
2. **Audio Processors:** Фильтры, Дилеи, Реверберация, Дисторшн.
3. **Control Signals (CV):** Константные значения, огибающие (ADSR), квантователи нот.
4. **Logic & Math:** Операторы сравнения, умножения, суммирования и Евклидовы алгоритмы.
5. **IO:** Выход на колонки (`Tone.getDestination`), MIDI-входы.

## 3. Математика и Технические Решения

### 3.1 Евклидов алгоритм (Bresenham-based)

**Строки: 542-618**

Логический узел `logic_euclidean` рассчитывает ритм в реальном времени внутри аудио-блока. Формула определения удара на шаге $i$ при общем количестве шагов $S$ и импульсов $P$:
$$Hit(i) = ((i + Rotate) \times P) \pmod S < P$$
Этот алгоритм гарантирует максимально равномерное распределение импульсов, превращаясь в "золотое сечение" ритмики.

### 3.2 JIT Скриптинг в аудио-потоке

**Строки: 61-91, 747-763**

Движок создает изолированную функцию из строки кода пользователя:
```javascript
const factory = new Function('memory', 'console', code);
```
Это позволяет реализовать любую математическую функцию (например, `Math.sin(val)`) непосредственно над аудио-сигналом с частотой дискретизации 44100Гц.

### 3.3 Динамическая перекоммутация (Edge Rebuild)

**Строки: 767-825**

При изменении связи между узлами движок:
1. Отключает все узлы (`disconnect`) для предотвращения утечек и обратных связей.
2. Сопоставляет ID порта (`sourceHandle` / `targetHandle`) с конкретным аудио-параметром.
3. Устанавливает новое соединение `source.connect(target)`.
