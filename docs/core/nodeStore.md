# 📘 АУДИТ: nodeStore.ts

## 1. Глобальное Описание

### Назначение
`nodeStore.ts` — это декларативное описание всех узлов и связей для **Node Editor**. Стор построен на базе `Zustand` и интегрирован с `React Flow`. Он служит "чертежом" для `GraphEngine`, храня информацию о типах узлов, их портах (входах/выходах), текущих параметрах и кастомных скриптах. Именно здесь определена вся библиотека модулей Studio Elite — от простых осцилляторов до продвинутых AI-генераторов текстур.

### Ключевые Подсистемы
- **Node Definitions (NODE_DEFS):** Обширный справочник всех доступных типов узлов со спецификацией их портов и начальных параметров.
- **Context Management:** Разделение графов на контексты (Poly, FX, Note), что позволяет иметь разные модульные среды внутри одного проекта.
- **Serialization System:** Средства для сохранения (`saveDevice`) и загрузки (`loadDevice`) созданных модульных патчей в формате JSON.
- **Param Tracking:** Система реактивного обновления параметров, которая мгновенно передает изменения из UI в аудио-движок.

## 2. Категории и Типы Узлов

**Строки: 85-356**

Модульная библиотека разделена на логические группы:
1. **Audio (Синий):** Базовые кирпичики синтеза (Osc, Filter, Env, LFO, Noise, VCA).
2. **Logic (Зеленый):** Операторы для сигналов управления (Math Op, Clock, Compare, Euclidean).
3. **Note (Желтый):** Обработка музыкальных событий (Quantizer, Chord, Arpeggiator).
4. **IO (Красный):** Входы и выходы аудио/MIDI.
5. **Advanced (Фиолетовый):** JIT-скриптинг (`script_js`) и интеграция с нейросетями (`ai_gen`).

## 3. Математика и Технические Решения

### 3.1 Система типизации Портов (NodePort)

**Строки: 39-43**

Каждое соединение имеет свой тип данных:
- `audio`: Стерео поток 44100Гц (для звука).
- `signal`: Контрольное напряжение (CV) для модуляции параметров.
- `data`: Массивы данных или URL-адреса (например, для AI текстур).
- `phase`: Фазовый сигнал (0-1) для сверхточном синхронизации осцилляторов и секвенсоров.

### 3.2 Управление состоянием React Flow

**Строки: 369-372**

Стор инкапсулирует стандартные обработчики `onNodesChange` и `onEdgesChange`, используя иммутабельные паттерны `applyNodeChanges`. Это гарантирует, что любое перетаскивание узла в UI корректно отразится в стейте без нарушения целостности данных.

### 3.3 Структура Script-узлов

**Строки: 323-343**

Для узлов `script_js` в сторе хранится шаблон кода (JSFX-style), который включает функции `init` и `process`. Это позволяет пользователю писать полноценные DSP-алгоритмы прямо в интерфейсе браузера.
