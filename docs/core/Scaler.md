# 📘 АУДИТ: Scaler.ts

## 1. Глобальное Описание

### Назначение
`Scaler.ts` — это математическое ядро теории музыки в Studio Elite 5.0. Модуль инкапсулирует логику работы с музыкальными ладами, аккордами и квантованием нот. Используя мощную библиотеку `@tonaljs/tonal`, он обеспечивает гармоническую чистоту звучания всех инструментов. Основная задача модуля — гарантировать, что любая сгенерированная или введенная пользователем нота соответствует выбранной тональности и ладу проекта.

### Ключевые Подсистемы
- **Scale Navigator:** Поиск ступени ноты в заданном ладу.
- **Chord Engine:** Алгоритмическое построение аккордов (трезвучия, септаккорды и др.) на основе выбранной ступени лада.
- **Pitch Quantizer (Snapping):** Принудительное "притягивание" хроматических нот к ближайшим нотам выбранного лада.

## 2. Алгоритмы Музыкальной Теории

### 2.1 Квантование (Snap to Scale)
**Строки: 72-91**
Когда поступает "неправильная" нота (например, Фа-диез в До-мажоре), метод `snapToScale` находит ближайшую разрешенную ноту.
Алгоритм:
1. Вычисляется высота ноты в рамках одной октавы ($Midi \pmod{12}$).
2. Рассчитывается минимальное расстояние до каждой ноты лада (с учетом циклической природы октавы).
3. Возвращается модифицированное MIDI-значение.
$$Distance(a, b) = \min(|a - b|, 12 - |a - b|)$$

### 2.2 Генератор аккордов
**Строки: 22-67**
Для любого уровня сложности (Triad, 7th, Power) модуль строит аккорд, шагая по ступеням выбранного лада. Например, для трезвучия (`triad`) берутся 1-я, 3-я и 5-я ступени относительно базовой ноты. Это гарантирует, что даже сложные аккорды будут звучать "в тональности", сохраняя минорный или мажорный характер в зависимости от контекста.

## 3. Математика и Технические Решения

### 3.1 Расчет октавных смещений
**Строки: 60-66**
При построении аккорда индекс ступени может выйти за пределы одной октавы ($Index \ge ScaleLength$). Скейлер корректно обрабатывает это через деление по модулю и расчет смещения октавы:
$$OctaveOffset = \lfloor Index / ScaleLength \rfloor$$
Это позволяет строить аккорды, выходящие за границы одной октавы, сохраняя правильные интервалы.

### 3.2 Поддержка типов аккордов
Модуль поддерживает следующие пресеты структур:
- **Triad:** [0, 2, 4] (Ступени 1-3-5)
- **7th:** [0, 2, 4, 6] (Ступени 1-3-5-7)
- **Power:** [0, 4, 7] (Корень, чистая квинта, октава)
- **Sus2/Sus4:** Замена терции на вторую или четвертую ступени.
