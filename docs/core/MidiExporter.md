# 📘 АУДИТ: MidiExporter.ts

## 1. Глобальное Описание

### Назначение
`MidiExporter.ts` — это модуль для преобразования музыкальных данных приложения в промышленный стандарт MIDI. Он позволяет пользователю "забирать" результаты своего творчества из Studio Elite 5.0 во внешние DAW (Ableton, FL Studio, Logic) для дальнейшей аранжировки и сведения. Модуль поддерживает экспорт отдельных дорожек: ударных, баса, секвенсоров (ML-185, Snake), пэдов и даже вероятностных последовательностей машины Тьюринга.

### Ключевые Подсистемы
- **MIDI-JS Integration:** Использование библиотеки `midi-writer-js` для формирования бинарного `.mid` файла.
- **Instrument Translators:** Набор функций-мапперов, которые переводят внутренние структуры данных (например, `BassStep` или `Stage`) в события MIDI-нот.
- **Rhythm Simulation:** Эмуляция работы пошаговых секвенсоров (ML-185) для корректного переноса длительностей и пауз.
- **Turing Logic Export:** Алгоритмическая генерация последовательности нот на основе текущего состояния сдвигового регистра.

## 2. Логика Экспорта по Инструментам

**Строки: 22-225**

Каждый целевой инструмент (`target`) обрабатывается индивидуально:
- **Drums:** Переводит 16-шаговую сетку в MIDI-ноты (`C1` — бочка, `D1` — малый и т.д.). Экспортируется цикл на 4 такта.
- **Bass:** Учитывает флаги `accent` (повышает Velocity до 127) и `slide` (увеличивает длительность до 1/8 для создания легато).
- **ML-185:** Самый сложный экспорт. Рассчитывает длительность ноты как произведение `length * pulseCount` и учитывает режимы гейта (`gateMode`).
- **Pads:** Экспортирует аккорды как длинные "целые" ноты (`duration: '1'`) на протяжении всей дорожки.
- **Turing Machine:** Симулирует 64 шага работы регистра, мапируя биты на высоту тона и выполняя побитовые сдвиги для создания вариативности.

## 3. Математика и Технические Решения

### 3.1 Конвертация частот в имена нот

**Строки: 90, 182**

Поскольку MIDIWriter работает с буквенными обозначениями нот (например, "C3"), используется библиотека `Tone.js` для перевода MIDI-номеров или частот в человекочитаемый формат:
$$NoteName = \text{Tone.Frequency}(MidiNumber, \text{'midi'}).\text{toNote}()$$

### 3.2 Математика регистра Тьюринга при экспорте

**Строки: 203-222**

Чтобы экспортировать 64 шага, не имея истории в сторе, модуль симулирует работу регистра:
1. Берет текущий `turingRegister`.
2. Вычисляет ноту: $Index = Reg \pmod{16}$.
3. Выполняет мутацию регистра:
   - Извлекает старший бит: $Bit = (Reg \gg (Bits - 1)) \& 1$.
   - Сдвигает регистр и вставляет новый бит: $Reg = ((Reg \ll 1) | \text{newBit}) \& \text{mask}$.
Это гарантирует, что экспортируемый файл будет звучать в стиле алгоритма машины Тьюринга.
