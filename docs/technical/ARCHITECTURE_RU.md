# 🏗 Архитектура Системы Studio Elite

Studio Elite — это сложное одностраничное веб-приложение (SPA), работающее на стыке React, WebGL и Web Audio API.

## 1. Технологический Стек (Golden Triangle)

| Слой | Технология | Описание |
|:---|:---|:---|
| **Audio** | **Tone.js** | Обертка над Web Audio API. Граф аудио-нод, сэмплирование, тайминг. |
| **Visual** | **R3F (Three.js)** | Декларативный 3D рендеринг. Компоненты сцены, освещение, шейдеры. |
| **State** | **Zustand** | Атомарное управление состоянием. Связующее звено. |

## 2. Управление Состоянием (Store Layer)
Состояние приложения разделено на домены (`src/store`):
- **`audioStore`**: Глобальный транспорт (Play/Stop, BPM), мастер-громкость, инициализация движка.
- **`visualStore`**: Состояние камеры, текущий фокус, активные оверлеи 2D HUD.
- **`themeStore`**: Управление цветами, пресетами оформления, генерация палитр.
- **`nodeStore`**: Хранение графа нод (узлы, соединения) для модульного редактора.
- **`instrumentStore`**: Параметры конкретных инструментов (Drums, Bass, Harmony).

## 3. AudioVisualBridge
Критически важный компонент (`src/lib/AudioVisualBridge.ts`).
- **Проблема**: React State обновляется слишком медленно для аудио (16ms vs 0ms) и вызывает ре-рендеры.
- **Решение**: Bridge использует систему событий (Event Emitter) и прямые ссылки (Refs) для синхронизации.
  - Аудио-поток (Tone.js) посылает событие `beat`.
  - Визуальный поток (useFrame) читает данные из Bridge и обновляет 3D-объекты напрямую, минуя React Render.

## 4. Файловая Структура
- `src/logic`: "Мозги" приложения. Чистые функции JS/TS (алгоритмы генерации, маппинг нот).
- `src/components/WebGL`: Все 3D компоненты.
- `src/components/HUD`: 2D интерфейсы (Overlay, Node Editor, Arrangement).

## 5. Оптимизация
- **Lazy Init**: Аудио-движок стартует только по клику юзера (обход iOS Silent Switch).
- **Instance Mesh**: Использование инстансирования для тысяч частиц (Drone Engine).
- **Throttling**: Ограничение частоты обновлений React-компонентов.
