# 🧮 Алгоритмическое Ядро Studio Elite

В этом документе описаны математические модели и алгоритмы, используемые для генерации музыки в Studio Elite.

## 1. Генератор Баса (Acid Core)

Основан на логике устройства **"Sting by Iftah"**.

### Параметры Генерации
Алгоритм использует два мета-параметра для создания бесконечных вариаций TB-303 линий:

#### **Density (Плотность) [0.0 - 1.0]**
Определяет ритмическую насыщенность.
- Генерируется **Seed (Зерно)**: массив из 16 случайных весов ($w_i \in [0, 1]$).
- Шаг $i$ активируется, если $w_i < Density$.
- **Результат**: При вращении ручки Density ноты добавляются/исчезают предсказуемо, сохраняя структуру паттерна (ноты не "прыгают" случайно).

#### **Type (Характер) [0.0 - 1.0]**
Определяет мелодическую гармонию.
- **0% - 30% (Tonal)**: Строго по ступеням трезвучия (Root, 3rd, 5th). Стабильный бас.
- **30% - 70% (Diatonic)**: Используются все ступени лада (например, фригийский).
- **70% - 100% (Chrom/Acid)**: Вводятся хроматизмы, тритоны и случайные скачки. Атональный хаос, характерный для Acid Techno.

#### **Акценты и Слайды**
- **Sldes**: Слайд (Slide) генерируется только если следующий шаг активен. Это обеспечивает корректное легато (портаменто).
- **Accents**: Вероятность акцента выше на слабых долях (off-beat) для создания грува.

## 2. Эвклидовы Ритмы (Drums)

Используется **Алгоритм Бьоркланда** (Bjorklund's algorithm) для распределения ударов.

### Логика
Задача: Равномерно распределить $k$ ударов (pulses) на $n$ шагах (steps).
Это позволяет создавать полиритмические структуры.

**Пример:** 5 ударов на 13 шагов ($E(5, 13)$) -> `[1, 0, 0, 1, 0, 1, 0, 0, 1, 0, 1, 0, 0]`

### Полиметрия
Каждый инструмент (Kick, Snare, HiHat) имеет независимую длину цикла ($n$).
- Если Kick = 16 шагов, а HiHat = 15 шагов, их паттерны будут смещаться друг относительно друга ("Phasing"), создавая длинный развивающийся ритм.

## 3. Секвенсор ML-185 (Stage Sequencer)

Не линейный, а сценический секвенсор.

### Понятие "Стадии" (Stage)
Вместо фиксированных шагов 1/16, секвенсор имеет 8 "Стадий".
У каждой стадии есть параметры:
- **Pitch**: Высота тона.
- **Length**: Длительность (количество тиков 1/16).
- **Gate Mode**: Режим срабатывания.
  - `Mute`: Пауза.
  - `Single`: Один триггер в начале.
  - `Multi`: Повторный триггер (Ratchet) на каждом шаге длительности.
  - `Hold`: Легато (не выключает ноту).
- **Probability**: Вероятность проигрывания стадии.

## 4. MDD Snake (Spatial Grid)

Матричный секвенсор 4x4, где мелодия определяется "Путём Змеи".

### Паттерны Движения
Голова секвенсора движется по сетке согласно алгоритму:
- **Linear**: Слева направо, сверху вниз (0, 1, 2... 15).
- **Zig-Zag**: Змейкой (0-3 вправо, 4-7 влево...).
- **Spiral**: По спирали от края к центру.
- **Random**: Случайный прыжок (Броуновское движение).

### Интеграция
- **Ритм** задает ML-185.
- **Высоту тона** берет MDD Snake (из текущей ячейки).
Если длина цикла ML-185 (сумма шагов) не равна 16, мелодия Snake будет смещаться относительно ритма.

## 5. Аудио Архитектура (Synth Stack)

### Acid Synth
- **Oscillator**: Sawtooth / Square (с Pulse Width Modulation).
- **Filter**: Diode Ladder Lowpass (18dB/oct).
- **Distortion**: Chebyshev Waveshaper для насыщения гармониками.

### Tone.js Optimizations
- Использование `Tone.Draw` для разделения аудио и UI потоков.
- Предзагрузка семплов в буфер перед стартом.
