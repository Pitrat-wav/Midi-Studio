# 📘 АУДИТ: useHandTracking.ts

## 1. Глобальное Описание

### Назначение
Этот хук реализует систему бесконтактного управления 3D-студией с помощью нейросетевого анализа видеопотока (AI Vision). Он использует библиотеку **MediaPipe**, чтобы превратить веб-камеру пользователя в пространственный контроллер. Хук отслеживает положение рук и пальцев, позволяя "дирижировать" музыкой или взаимодействовать с виртуальными объектами в пространстве.

### Связи
- **AI Engine:** Использует `@mediapipe/tasks-vision` для распознавания лендмарков рук.
- **Store:** Передает координаты рук в `visualStore` для последующей отрисовки в 3D и управления параметрами.
- **Hardware:** Запрашивает доступ к видеокамере через WebRTC API (`getUserMedia`).

---

## 2. Интерфейс и UI Элементы

Хук создает и управляет следующим элементом:
- **Video PIP (Picture-in-Picture):** Небольшое плавающее окно предпросмотра с камеры в углу экрана (320x240 пикселей). 
- Окно зеркально отражено (`scaleX(-1)`) и имеет фирменную рамку цвета Telegram (`#3390ec`), что обеспечивает удобную обратную связь — пользователь видит, какие именно жесты считывает система.

## 3. Логика и Математика

### 3.1 Инициализация нейросети (MediaPipe)

**Строки:** 32-44

Система загружает WASM-модули и обученную модель `hand_landmarker`. Использование GPU ускорения (`delegate: "GPU"`) позволяет проводить анализ в реальном времени с минимальной задержкой.

### 3.2 Распознавание лендмарков (Predict Loop)

**Строки:** 79-96

Функция `predict` работает внутри цикла `requestAnimationFrame`, обеспечивая максимально возможную частоту кадров (до 60 FPS).

**Математическая модель данных:**
Результатом работы нейросети является массив из 21 точки (Landmarks) для каждой руки.
Каждая точка $P_i$ имеет нормализованные координаты:
$$P_i = \{x, y, z\} \in [0, 1]$$
Где:
- $x, y$: Положение в плоскости кадра.
- $z$: Глубина (расстояние от камеры), рассчитываемая алгоритмом на основе размера ладони.

### 3.3 Пространственное управление

Полученные координаты передаются в `visualStore`. В дальнейшем в 3D сцене они могут быть преобразованы из экранного пространства камеры в мировое 3D пространство для создания эффекта "виртуального касания" инструментов.
