# usePoseTracking.ts

### 1. Глобальное Описание (File Overview)
* **Назначение:** Менеджер трекинга поз (Skeleton Tracking) на базе AI MediaPipe. Захватывает видеопоток с веб-камеры, анализирует положение тела пользователя и передает координаты суставов в визуальную систему для интерактивности.
* **Связи:** 
    * Импортирует: `@mediapipe/tasks-vision`, `useVisualStore`.
    * Влияет на: Визуализаторы, поддерживающие интерактивность через движения (ID 51-60 и др.).

### 2. Интерфейс и UI Элементы (Visual Elements)

* **Элемент:** Видео-превью (Webcam Feedback)
    * **Где находится:** Строки 54-69.
    * **Логика:** Создает плавающий круглый элемент в углу экрана (bottom: 20px, right: 20px).
    * **Стили:** Пропорционально зеркальное отображение (`scaleX(-1)`), синяя рамка (#3390ec), размытость и тень.

### 3. Логика и Математика (Logic & Math)

* **Функция:** `initialize()`
* **Что делает:** Загружает WASM-библиотеки MediaPipe и инициализирует модель `pose_landmarker_lite`.
* **Математическая модель:**
    Используется модель со сниженной точностью (lite) для производительности на мобильных устройствах, работающая в режиме делегирования на GPU.

* **Функция:** `predict()` (Цикл обработки)
* **Построчный разбор:**
    1. Проверяет наличие видеопотока.
    2. Вызывает `detectForVideo(video, timestamp)`.
    3. Если найден скелет (landmarks), мапирует их в массив объектов с координатами (x, y, z) и видимостью.
    4. Обновляет стор через `setPoseData`.
    5. Зацикливает выполнение через `requestAnimationFrame`.

* **Математика Координат:**
    MediaPipe возвращает нормализованные координаты (0.0..1.0).
    $$ X_{screen} = X_{normalized} \times Width_{video} $$
    $$ Y_{screen} = Y_{normalized} \times Height_{video} $$
