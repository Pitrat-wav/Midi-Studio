# 📘 АУДИТ: useDeterministicEngine.ts

## 1. Глобальное Описание

### Назначение
Этот хук является "мостом" между React-интерфейсом и детерминированным музыкальным движком, работающим в фоновом потоке (Web Worker). Его основная задача — управление жизненным циклом воркера, который запускает **Pyodide** (Python в браузере) для сложной алгоритмической генерации музыкальных паттернов, не блокируя основной поток интерфейса.

### Связи
- **Worker:** Инициализирует и общается с `src/workers/DeterministicWorker.ts`.
- **Logic:** Служит интерфейсом для вызова алгоритмов генерации мелодий и ритмов на языке Python.
- **Использование:** Применяется в `CompositionManager.ts` для создания новых композиций по запросу пользователя.

---

## 2. Интерфейс и UI Элементы

Хук предоставляет следующие состояния для UI:
- **isReady:** Флаг готовности движка Python. Пока он равен `false`, кнопки генерации должны быть заблокированы.
- **lastResult:** Объект с последним сгенерированным паттерном (ноты баса, барабаны и т.д.).
- **generatePattern():** Функция-триггер для запуска процесса генерации.

## 3. Логика и Математика

### 3.1 Архитектура фоновой обработки

**Строки:** 12-31

Хук использует Web Worker для изоляции тяжелых вычислений Python.
1. **Инициализация:** При первом рендере создается новый экземпляр воркера.
2. **Тип сообщения (INIT):** Отправляет сигнал воркеру начать загрузку среды Pyodide и необходимых Python-пакетов.
3. **Обработка ответа:** Слушает события `onmessage`. 
   - `READY` — сигнализирует о полной загрузке среды.
   - `RESULT` — содержит готовые музыкальные данные.

### 3.2 Генерация (GENERATE)

**Строки:** 33-39

При вызове функции `generatePattern` в воркер отправляется пакет данных с параметрами тональности и лада.

**Математическая модель данных:**
Движок на стороне Python (в воркере) получает:
$$Input = \{Scale, Root\}$$
И возвращает детерминированный массив (обычно 16-32 шага), рассчитанный с использованием специализированных музыкальных библиотек Python.
