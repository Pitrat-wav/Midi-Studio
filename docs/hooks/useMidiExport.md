# 📘 АУДИТ: useMidiExport.ts

## 1. Глобальное Описание

### Назначение
Этот хук реализует функционал экспорта созданных музыкальных паттернов в стандартный MIDI-формат. Он является связующим звеном между состоянием инструментов (Stores) и логикой формирования двоичного файла. Уникальной особенностью является глубокая интеграция с **Telegram WebApp**, позволяющая мгновенно отправлять сгенерированный MIDI-файл прямо пользователю в чат через внешний API.

### Связи
- **Stores:** Считывает данные изо всех магазинов инструментов (Drums, Bass, Sequencer, Harmony, Pads).
- **Logic:** Использует `MidiExporter.ts` для преобразования паттернов в байты MIDI и `bjorklund.ts` для пересчета эвклидовых ритмов "на лету" перед экспортом.
- **API:** Взаимодействует с бэкэндом через `VITE_API_URL` для загрузки и доставки файла.

---

## 2. Интерфейс и UI Элементы

Хук предоставляет:
- **exportMidi(target):** Функция, принимающая цель экспорта (конкретный инструмент или секвенция).
- **isExporting:** Состояние загрузки, позволяющее отображать индикатор прогресса (Spinner) в UI.
- **Telegram Alerts:** Выводит подтверждение (`showAlert`) об успешной отправке файла или ошибке.

## 3. Логика и Математика

### 3.1 Квантование данных перед экспортом

**Строки:** 24-30

Для барабанов хук заново генерирует массивы шагов на основе эвклидовых параметров (Steps/Pulses) с помощью алгоритма Бьёрклунда. Это гарантирует, что в MIDI-файл попадет именно тот ритм, который пользователь слышит в данный момент.

### 3.2 Формирование MIDI-контейнера

**Строки:** 32-45

Вызов `exportToMidi` собирает все параметры:
- **BPM:** Устанавливает темп проекта в заголовке MIDI-файла.
- **Patterns:** Передает ноты и триггеры.
- **Smart Chord:** Если включено, ноты секвенсора автоматически расширяются до аккордов перед записью.
- **Turing Data:** Текущее состояние регистра сдвига преобразуется в мелодическую линию.

### 3.3 Кодирование и передача (Base64)

**Строки:** 46-57

Двоичные данные MIDI преобразуются в строку Base64 для передачи внутри JSON через HTTP POST запрос:
$$Base64 = \text{btoa}(\text{String.fromCharCode}(...Bytes))$$
Это стандартный способ передачи бинарных файлов в веб-интерфейсах.
