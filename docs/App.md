# App.tsx

### 1. Глобальное Описание (File Overview)
* **Назначение:** Главный входной компонент приложения (Root Component). Отвечает за инициализацию аудио-движка, рендеринг WebGL-сцены, управление HUD-overlays и навигацию между видами (3D, Nodes, Live, Arrange).
* **Связи:** 
    * Импортирует: Все основные сторы (`audioStore`, `visualStore`, `nodeStore`), WebGLScene, HUD экраны всех инструментов.
    * Инициализирует: `GamepadManager`, `GraphEngine`, `AudioVisualBridge`.

### 2. Интерфейс и UI Элементы (Visual Elements)

* **Элемент:** Экран инициализации (Init Screen)
    * **Где находится:** Строки 150-177.
    * **Логика:** Отображается до тех пор, пока аудио-движок не будет запущен вручную (требование браузеров для звука).

* **Элемент:** Кнопка "Launch Studio"
    * **Где находится:** Строки 158-163.
    * **Логика:** Вызывает асинхронную функцию `handleInit`.

* **Элемент:** Transport Controls (Play/Stop/BPM)
    * **Где находится:** Строки 204–234.
    * **Логика:** 2D-оверлей для управления воспроизведением. Слайдер BPM меняет значение в `audioStore`.

* **Элемент:** Навигационный бар (Instrument Navigation)
    * **Где находится:** Строки 272-278.
    * **Логика:** Голографическое меню для быстрого переключения между инструментами.

### 3. Логика и Математика (Logic & Math)

* **Функция:** `handleInit()`
* **Что делает:** Запускает процесс инициализации аудио-контекста.
* **Математическая модель:** 
    Инициализация — это переход из состояния `Suspended` в `Running`.
    $$ ContextState: \text{Suspended} \xrightarrow{UserAction} \text{Running} $$

* **Построчный разбор (`useEffect` синхронизации):**
    1. **Синхронизация Роутинга:** При изменении `edges` (связей в нодах), вызывает `recalculateRouting()`.
    2. **Авто-скрытие HUD:** Если приложение запущено и играет музыка, через 3000 мс HUD скрывается (`setShowOverlay(false)`).
    3. **BPM Sync:** При изменении темпа в сторе, значение прокидывается в `AudioVisualBridge` для синхронизации визуальных эффектов с тактом.

* **Условный рендеринг видов (`activeView`):**
    Использует `switch`-подобную логику для отображения:
    * `VISUALIZER` -> `VisualEngine`
    * `NODES` -> `NodeEditor`
    * `3D` -> `WebGLScene`
