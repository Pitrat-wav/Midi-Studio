# 📘 АУДИТ: AudioVisualBridge.ts

## 1. Глобальное Описание

### Назначение
Этот файл является "мостом" между миром звука (Tone.js) и миром графики (Three.js/WebGL). Он отвечает за то, чтобы 3D-объекты реагировали на музыку в реальном времени. Модуль анализирует звуковую волну, выделяет частоты (басы, средние, высокие) и преобразует их в данные, которые могут использовать шейдеры и 3D-компоненты для анимации.

### Связи
- **Импорты:** `tone` (аудио-источник), `useVisualStore` (стейт-хранилище).
- **Используется в:** Почти во всех WebGL компонентах через хук `useAudioVisualBridge()`.
- **Архитектура:** Реализован как **Singleton** (одиночный экземпляр), обеспечивая единую точку правды для всей визуализации.

## 3. Логика и Математика

### 3.1 Функция: `startUpdateLoop()`

**Строки:** 122-186

**Что делает:**  
Запускает бесконечный цикл обновления (60 раз в секунду), который считывает текущие аудио-данные и рассчитывает параметры для визуализации.

**Математическая модель (RMS - Среднеквадратичная громкость):**

Для вычисления общей интенсивности звука берется массив выборок звуковой волны $x_i$ и рассчитывается RMS:

$$RMS = \sqrt{\frac{1}{N} \sum_{i=0}^{N-1} x_i^2}$$

Где $N$ — количество образцов (в коде 256). Это значение передается в шейдеры как `uAudioIntensity`.

**Частотный анализ (FFT):**
Спектр из 512 полос разделяется на три диапазона:
1. **Low (Низкие):** Первые 10% полос (~0-2 кГц).
2. **Mid (Средние):** От 10% до 40% полос (~2-8 кГц).
3. **High (Высокие):** Остальные полосы (> 8 кГц).

Для каждого диапазона вычисляется среднее арифметическое значение амплитуды.

### 3.2 Функция: `triggerMIDI(event, time)`

**Строки:** 231-259

**Что делает:**  
Синхронизирует вспышки на экране с моментом извлечения звука.

**Математическая модель (Тайминг):**
Используется механизм **Tone.Draw**, который позволяет запланировать выполнение JavaScript-кода точно в момент, когда аудио-событие дойдет до динамиков.

**Построчный разбор:**
1. Получает MIDI-событие (нота нажата/отпущена).
2. Вызывает `Tone.Draw.schedule`, передавая точное время `actualTime`.
3. Внутри функции обновления устанавливает "пульс" (`triggerPulse`) для конкретного инструмента:
   - Если бочка (kick) -> `pulses.kick = velocity`.
   - Если нота синта -> `pulses.note = velocity`.

---

### 3.3 Система "Затухания" (Decay)

**Строки:** 166-168

В каждом кадре значения "пульсов" уменьшаются, чтобы вспышка была короткой:
$$Pulse_{t+1} = \max(0, Pulse_t - 0.05)$$
