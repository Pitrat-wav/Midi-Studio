# 📘 АУДИТ: GamepadManager.ts

## 1. Глобальное Описание

### Назначение
Этот файл реализует систему управления студией с помощью игровых контроллеров (геймпадов), с особой оптимизацией под **Sony PlayStation 5 DualSense**. Менеджер отслеживает подключение устройств, мапит физические кнопки на музыкальные команды и управляет тактильной отдачей (вибрацией).

### Связи
- **Импорты:** Взаимодействует с `audioStore` (громкость, BPM, Play/Stop) и `visualStore` (камера, фокус, пресеты).
- **События:** Слушает системные события браузера `gamepadconnected` и `gamepaddisconnected`.

---

## 2. Интерфейс и UI Элементы

### Маппинг кнопок (DualSense / Standard)
Этот файл превращает геймпад в пульт управления:
- **Кнопка "Крест" (X):** Запуск/Остановка секвенсора (`togglePlay`).
- **Кнопка "Круг" (O):** Заглушить (Mute) текущий сфокусированный инструмент.
- **Кнопка "Треугольник" (Δ):** Режим ПАНИКА — немедленная остановка всего звука.
- **Кнопка "Квадрат" (□):** Переключение камеры между общим видом и мастером.
- **L1 / R1:** Циклическое переключение между инструментами (движение камеры).
- **D-Pad (Стрелки):** Регулировка BPM и смена визуальных пресетов неба.

## 3. Логика и Математика

### 3.1 Система опроса (Polling)

**Строки:** 41-47

Браузерная API геймпадов не работает через события для стиков, поэтому используется цикл `requestAnimationFrame`, который опрашивает состояние устройства каждый кадр (60 FPS).

---

### 3.2 Математика стиков и Мертвая зона (Deadzone)

**Строки:** 79-95

Чтобы камера или параметры не "плыли" из-за износа стиков контроллера, применяется алгоритм мертвой зоны:

$$Value_{final} = \begin{cases} 
Value_{raw} & \text{если } |Value_{raw}| > 0.1 \\
0 & \text{если } |Value_{raw}| \leq 0.1 
\end{cases}$$

Это гарантирует, что микро-движения стика в центральном положении будут игнорироваться.

---

### 3.3 Регулировка BPM правым стиком

**Строки:** 68-74

Правый стик (ось Y) используется как чувствительный слайдер для темпа:
$$BPM_{new} = \text{clamp}(60, 240, BPM_{old} - Y_{stick} \times 2)$$

Чувствительность умножается на 2 для более быстрого отклика.

---

### 3.4 Тактильная отдача (Vibration)

**Строки:** 153-163

Функция `vibrate` использует Dual-Rumble эффект. Она принимает длительность и интенсивность, создавая физическое подтверждение действий пользователя (например, сильная вибрация при нажатии "Panic").

**Параметры:**
- `weakMagnitude`: Вибрация высокочастотных моторов.
- `strongMagnitude`: Вибрация низкочастотных (грубых) моторов.
