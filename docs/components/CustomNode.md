# 📘 АУДИТ: CustomNode.tsx

## 1. Глобальное Описание

### Назначение
Этот компонент является фундаментальным визуальным блоком модульного движка (Graph Engine). Каждый узел представляет собой определенную аудио-функцию (осциллятор, фильтр), логическую операцию (сравнение, математику) или скрипт. С помощью этих узлов пользователь может собирать собственные синтезаторы и алгоритмы напрямую в интерфейсе, соединяя входы и выходы виртуальными кабелями.

### Связи
- **React Flow:** Встраивается в систему как кастомный тип узла.
- **Store:** Активно использует `useNodeStore` для обновления параметров и состояния графа.
- **Logic:** Взаимодействует с `GraphEngine.ts` для получения данных осциллограмм и управления аудио-нодами в реальном времени.

## 2. Интерфейс и UI Элементы

### Анатомия узла
* **Header:** Название узла, иконка категории (Audio, Script, AI, Logic) и кнопка удаления.
* **Ports (Handles):** Входы (Left) и выходы (Right). Цвет порта отражает тип данных (Audio, Control, MIDI).
* **Control Bay:** Динамически меняющиеся ползунки (Range), выпадающие списки (Select) и текстовые поля в зависимости от типа узла.
* **Canvas Visualizer:** Миниатюрный монитор (Scope), отображающий волну в реальном времени для осцилляторов и скриптовых узлов.
* **Script Editor:** Всплывающее окно для написания JS-кода (JSFX-style) прямо внутри узла.

## 3. Логика и Математика

### 3.1 Отрисовка осциллограмм (Visual Scope)

**Строки:** 17-78

Каждый узел может иметь собственный графический цикл `requestAnimationFrame`.
Для типа `visual_scope` данные берутся напрямую из аудио-буфера через `Tone.Waveform`.
**Математика отрисовки:**
1. Получается массив значений амплитуд $V$ (Float32Array).
2. Шаг выборки $Step = \text{ceil}(Length_V / Width_{canvas})$.
3. Для каждого пикселя $x \in [0, 140]$:
   $$y = 30 - (V_{x \times Step} \times 28)$$
   Где 30 — центр холста, а 28 — коэффициент усиления для визуализации.

### 3.2 Динамическая параметризация

**Строки:** 89-216

Компонент анализирует объект `data.params` и строит интерфейс:
- **Enum (Выпадающие списки):** Если ключ совпадает с предопределенными (type, mode, op), предлагается выбор из музыкальных и логических констант.
- **Number (Ползунки):** Используется логарифмическая или линейная шкала (например, до 2000 Гц для частоты).
- **Arrays (Секвенсор):** Отрисовывается мини-секция слайдеров для каждой ступени паттерна.
