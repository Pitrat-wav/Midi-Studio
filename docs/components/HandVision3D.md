# 📘 АУДИТ: HandVision3D.tsx

## 1. Глобальное Описание

### Назначение
Этот компонент является визуальным и логическим мостом между системой AI-трекинга рук (`useHandTracking`) и 3D-миром студии. Он отрисовывает виртуальный скелет руки в пространстве и обрабатывает пространственные жесты (например, щипок — "pinch"), превращая движения пальцев в сигналы управления синтезаторами. Это реализует концепцию "Терменвокса" (Theremin), позволяя управлять громкостью или фильтром звука, просто перемещая руку в воздухе перед камерой.

### Связи
- **Visual Store:** Считывает лендмарки рук `handData`.
- **Audio Engine:** Напрямую манипулирует параметрами `Tone.js` узлов (HarmSynth, BassSynth) для минимизации задержки (low latency).
- **MediaPipe:** Визуализирует 21 точку суставов, распознанных нейросетью.

## 2. Интерфейс и UI Элементы

### Визуализация скелета
- **FingerJoint:** Каждая из 21 точек представлена сферой синего цвета (`#3390ec`).
- **Bones (Кости):** Соединительные линии между суставами для создания структуры руки.
- **Space Mapping:** Координаты с камеры масштабируются до размеров 20x16 единиц WebGL, что позволяет руке "заполнять" весь экран на определенной глубине.

## 3. Логика и Математика

### 3.1 Дешифрация жестов (Pinch Detection)

**Строки: 50-57**

Система анализирует расстояние между кончиком большого пальца (точка 4) и кончиком указательного пальца (точка 8).
$$Distance = \sqrt{(x_4-x_8)^2 + (y_4-y_8)^2 + (z_4-z_8)^2}$$
Если $Distance < 0.5$ единиц, жест считается активным ("щипок").

### 3.2 Пространственная модуляция (Theremin Logic)

**Строки: 58-87**

Когда жест "щипок" активен, положение руки в пространстве управляет звуком:
1. **Vertical (Y-axis):** Управляет громкостью или частотой среза фильтра (Cutoff).
   $$Height_{norm} = \text{clamp}(Y_{index} / 4 + 0.5, 0, 1)$$
   $$Cutoff = 40 + Height_{norm} \times 5000 \text{ Hz}$$
2. **Horizontal (X-axis):** Может быть назначено на панораму или другие параметры.

### 3.3 Оптимизация производительности

Для обеспечения плавности звука (без щелчков), изменения параметров применяются напрямую к узлам `Tone.js` с использованием метода `.rampTo(value, 0.05)`, что создает микро-сглаживание за 50 мс. Синхронизация с React-стором происходит с задержкой (throttling), чтобы не перегружать основной поток интерфейса постоянными обновлениями.
