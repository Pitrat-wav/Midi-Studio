# 📘 АУДИТ: Sequencer3D.tsx

## 1. Глобальное Описание

### Назначение
Компонент `Sequencer3D` (Sequencer Hub) является центральным узлом для управления логическими инструментами в 3D мире. Он визуализирует работу машины Тьюринга (Neural Sequencer) и предоставляет переключатели для модульных подсистем (Stages, Snake). Основной визуальный фокус направлен на "Нейронное кольцо", которое отображает состояние битов в сдвиговом регистре как сеть взаимосвязанных узлов и синапсов.

### Связи
- **Sequencer Store:** Источник данных для всех секвенсоров. Хранит состояние регистра Тьюринга, вероятность мутации и флаги активности.
- **ML185 / Snake:** Хотя эти модули вынесены в отдельные файлы, `Sequencer3D` служит их пространственным центром.
- **Spatial Layout:** Определяет положение хаба в общей сцене.

## 2. Интерфейс и UI Элементы

### Нейронное кольцо (Neural Ring)
**Строки: 23-113**
- **Nucleus (Ядро):** Розовый икосаэдр в центре, яркость которого зависит от параметра вероятности (`probability`). Символизирует "мозг" системы.
- **Neural Nodes:** 16 сфер, расположенных по кругу. Каждая сфера соответствует биту в регистре машины Тьюринга. Если бит активен (1), сфера ярко светится голубым цветом.
- **Synapses (Синапсы):** Линии, соединяющие узлы в кольцо. Толщина и яркость линии зависят от активности связанных узлов.
- **Energy Beams:** Тонкие розовые лучи, идущие от активных узлов к центральному ядру, визуализируя поток данных.

### Мастер-консоль управления
**Строки: 121-144**
Три 3D-кнопки (Button3D) для запуска соответствующих логических движков:
- **STAGES:** Запуск многоэтапного секвенсора.
- **SNAKE:** Запуск пространственного секвенсора "Змейка".
- **TURING:** Запуск вероятностной машины.

## 3. Логика и Математика

### 3.1 Построение нейронной сети (Trigonometric Positioning)

**Строки: 57-67**

Позиции узлов рассчитываются классическим методом распределения по кругу:
$$X_i = \cos(\frac{i}{16} \times 2\pi) \times Radius$$
$$Z_i = \sin(\frac{i}{16} \times 2\pi) \times Radius$$
Где $i \in [0, 15]$, а $Radius = 2.0$.

### 3.2 Визуализация регистра (Bitwise Mapping)

**Строки: 68**

Состояние каждой сферы определяется побитовым сдвигом и маскированием десятичного числа регистра:
$$isActive = (Register \gg i) \& 1$$
Это позволяет в реальном времени видеть, как число в памяти машины превращается в геометрический узор.

### 3.3 Динамическая пульсация (Scaling)

**Строки: 38-39**

Все кольцо медленно пульсирует в зависимости от времени, создавая эффект "дыхания" системы:
$$Scale = 1 + \sin(Time \times 2) \times 0.05$$
