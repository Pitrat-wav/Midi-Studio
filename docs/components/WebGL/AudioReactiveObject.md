# 📘 АУДИТ: AudioReactiveObject.tsx

## 1. Глобальное Описание

### Назначение
Этот компонент создает амбиентный 3D-объект в форме торуса (кольца), который служит визуальным фоном музыкальной студии. Его уникальность заключается в динамической реакции на звук и действия пользователя (Interaction Energy). Объект пульсирует в такт бочке (Kick) и меняет свои физические свойства (цвет, скорость деформации) при активном взаимодействии с интерфейсом.

### Связи
- **AudioVisual Bridge:** Получает сигналы пульсации от конкретных музыкальных инструментов (Kick).
- **Visual Store:** Считывает `interactionEnergy` — коэффициент активности пользователя.
- **WebGL Ecosystem:** Находится на заднем плане сцены (`position: [0, 0, -8]`).

---

## 2. Интерфейс и UI Элементы

### Визуальные свойства
- **Geometry:** Тонкий `torusGeometry` с высоким разрешением сегментов (128), что создает эффект плавного круга.
- **Material:** Использует `MeshWobbleMaterial` из библиотеки `@react-three/drei`. Этот материал позволяет объекту "волноваться" (wobble) как жидкость.

## 3. Логика и Математика

### 3.1 Расчет динамического масштабирования (Pulse Logic)

**Строки:** 17-23

Размер объекта рассчитывается в каждом кадре как сумма базового размера, аудио-пульса и энергии взаимодействия:
$$Scale = 1.0 + (Pulse_{kick} \times 0.3) + (Energy_{interaction} \times 0.5)$$

### 3.2 Изменение свойств материала

**Строки:** 33-37

Материал динамически меняет свои параметры в зависимости от активности пользователя:
- **Цвет:** Синий по умолчанию (`#3390ec`), красный при активности (`#ff3b30`).
- **Wobble Speed:** Чем выше активность, тем быстрее объект "вибрирует" ($1 + Energy \times 4$).
- **Wobble Factor:** Амплитуда деформации увеличивается вместе с энергией ($0.4 + Energy$).

### 3.3 Вращение (Active Rotation)

**Строки:** 25-27

При достижении порога активности `interactionEnergy > 0.1` объект начинает ускоренно вращаться по оси Z, добавляя динамики в сцену.
