# 📘 АУДИТ: AIPanel.tsx

## 1. Глобальное Описание

### Назначение
Этот компонент представляет собой интерфейс для работы с генеративными нейросетями. **AI Texture Lab** позволяет пользователю на лету изменять визуальный облик 3D-мира, генерируя новые текстуры с помощью модели Stable Diffusion по текстовому описанию. Сгенерированные текстуры автоматически накладываются на сферические объекты в 3D пространстве студии.

### Связи
- **Store:** Работает с `aiStore` для управления состоянием генерации и хранения API-ключей.
- **External API:** Взаимодействует с Hugging Face через Inference API.
- **Visuals:** Передает URL сгенерированной текстуры в 3D движок для обновления материалов.

## 2. Интерфейс и UI Элементы

### Структура панели
- **Floating Button:** Плавающая кнопка с иконкой волшебной палочки для вызова панели.
- **Key Setup:** Секция настройки API-ключа Hugging Face (скрывается после ввода).
- **Prompt Editor:** Текстовая область для ввода описания будущей текстуры.
- **Preview Area:** Окно предварительного просмотра сгенерированного изображения.
- **Status Indicators:** Отображение процесса генерации (Generating...) и возможных ошибок API.

---

## 3. Логика и Математика

### 3.1 Управление состоянием анимации

**Строки:** 33-100

Для появления и скрытия панели используется библиотека `framer-motion`. Применяются декларативные трансформации:
- **initial:** Состояние до появления (`opacity: 0`, на 20px правее).
- **animate:** Целевое состояние.
- **exit:** Состояние при закрытии (обеспечивает плавное исчезновение).

### 3.2 Режим генерации (Texture Mapping)

**Строки:** 14-17, 84-90

Процесс генерации асинхронен. После получения ссылки `currentTextureUrl`, компонент уведомляет пользователя о применении текстуры ко всем "планетам" (инструментам) в 3D сцене. Технически это достигается путем изменения `map` свойства материалов в Three.js.
