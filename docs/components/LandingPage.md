# 📘 АУДИТ: LandingPage.tsx

## 1. Глобальное Описание

### Назначение
Этот компонент является "посадочной страницей" (Splash Screen) приложения. Он выполняет две критические функции: во-первых, создает премиальное первое впечатление с помощью современной типографики и эффектов свечения; во-вторых, служит инициализатором аудио-движка. Нажатие на главную кнопку разблокирует AudioContext браузера, что необходимо для воспроизведения звука.

### Связи
- **Audio Engine:** Кнопка "INITIALIZE ENGINE" вызывает функцию `onLaunch`, которая инициализирует `Tone.js`.
- **Branding:** Задает визуальный тон всему приложению (Immersive Generative Architecture).

---

## 2. Интерфейс и UI Элементы

### Дизайн системы
- **Background Glow:** Динамические градиенты на фоне для создания ощущения глубины пространства.
- **Title Text:** Комбинация жирного начертания и акцентного цвета для бренда "AUTONOMOUS STUDIO".
- **Launch Button:** Высокотехнологичная кнопка с внутренним свечением (`btn-glow`) и состоянием наведения (`hover`).

### Информационные блоки
- **Feature Grid:** Список ключевых технологий (AI Vision, Pyodide, WebGL Shaders), представленный иконками и метками.
- **Shortcuts Bar:** Краткая шпаргалка по основным горячим клавишам прямо на главном экране.

---

## 3. Логика и Математика

### 3.1 Разблокировка аудио-контекста

Согласно политикам безопасности современных браузеров (Autoplay Policy), звук не может быть запущен автоматически без осознанного действия пользователя (клик, нажатие клавиши).
Компонент `LandingPage` является мостом, превращающим техническое требование безопасности в часть пользовательского опыта "загрузки системы" (System Initialization).

### 3.2 Визуальные эффекты (CSS-Logic)

Использование `isLaunchHovered` состояния позволяет анимировать не только саму кнопку, но и фоновое свечение вокруг нее, усиливая "притяжение" внимания к основному призыву к действию (CTA).
