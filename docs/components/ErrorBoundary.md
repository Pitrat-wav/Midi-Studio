# 📘 АУДИТ: ErrorBoundary.tsx

## 1. Глобальное Описание

### Назначение
Этот компонент является предохранителем (Error Boundary) для React-приложения. Он перехватывает ошибки JavaScript в дочерних компонентах, предотвращая "белый экран" и полный крах интерфейса. Вместо упавшего приложения пользователь видит стилизованное окно с информацией об ошибке и кнопками восстановления.

### Связи
- **Integration:** Оборачивает ключевые части приложения (например, `App`, `WebGLCanvas`, `HUD`).
- **Telegram WebApp:** Использует цвета из `tg-theme` для бесшовной интеграции с интерфейсом Telegram.

---

## 2. Интерфейс и UI Элементы

При возникновении ошибки отображается диагностическое окно:
- **Alert Icon:** Визуальный индикатор проблемы.
- **Fallback Name:** Название упавшего компонента (например, "Root App" или "WebGL Renderer").
- **Error Stack:** Техническое описание ошибки (в теге `pre`) для разработчиков.
- **Refresh Button:** Кнопка мягкой перезагрузки компонента (сброс локального состояния ошибки).
- **Hard Reset Button:** (Только для Root App) Полный сброс `sessionStorage` и перезагрузка страницы.

---

## 3. Логика и Математика

### 3.1 Жизненный цикл обработки ошибок

**Строки:** 14-30

Классовый компонент использует два ключевых метода React:
1. **getDerivedStateFromError:** Статический метод, который обновляет состояние `hasError: true` при возникновении исключения, что вызывает перерисовку на запасной интерфейс.
2. **componentDidCatch:** Используется для логирования ошибки в консоль (или внешнюю систему мониторинга).

### 3.2 Глубокий сброс (Hard Reset)

**Строки:** 32-37

При критических ошибках, связанных с повреждением данных в памяти, предусмотрен метод `handleDeepReset`:
1. Вызывается подтверждение `confirm`.
2. Очищается `sessionStorage.clear()`.
3. Выполняется принудительная перезагрузка страницы `window.location.reload()`.
