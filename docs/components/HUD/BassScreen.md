# 📘 АУДИТ: BassScreen.tsx

## 1. Глобальное Описание

### Назначение
Этот компонент представляет собой 2D-интерфейс управления басовым синтезатором, оформленный в стиле "Magazine HUD" (глянцевого архитектурного журнала). Он предоставляет полный контроль над двумя движками синтеза (Acid и FM), глобальной гармонией и ритмическим паттерном. Дизайн ориентирован на премиальную эстетику и удобство управления сложными параметрами звука.

### Связи
- **Bass Store:** Основной источник состояния для параметров синтеза и паттернов секвенсора.
- **Harmony Store:** Интеграция с глобальными настройками тональности (Root) и лада (Scale).
- **Visual Store:** Позволяет закрывать HUD через `setFocusInstrument(null)`.

## 2. Интерфейс и UI Элементы

### Выбор движка (Engine Selection)
**Строки: 32-49**
Позволяет переключаться между двумя типами звучания:
1. **ACID LIQUID:** Классический субтрактивный синтез с акцентом на резонанс фильтра.
2. **FM METALLIC:** Частотно-модуляционный синтез для создания индустриальных и металлических звуков.

### Редактор паттерна (Sequencer Editorial)
**Строки: 133-156**
Визуальный ряд из 16 вертикальных полос:
- **Высота полосы:** Отражает активность (`active`) и акцент (`accent`).
- **White Border Top:** Слайд (`slide`) — плавный переход между нотами.
- **Модификаторы:** Использование Shift (Accent) и Alt (Slide) для глубокого редактирования ритма.

## 3. Логика и Математика

### 3.1 Переключение параметров в зависимости от движка

**Строки: 89-131**

Интерфейс динамически меняет набор ползунков при смене движка:
- **Для Acid:** Cutoff (50-10000 Гц), Resonance (0.1-20), Morph и Distortion.
- **Для FM:** Harmonicity (коэффициент гармоник), Mod Index (глубина FM), Attack и Decay.

### 3.2 Управление паттерном (Bitwise-like Toggle)

**Строки: 11-16**

Функция `toggleStepParam` реализует неизменяемое обновление массива объектов. При клике на шаг создается копия паттерна, в которой инвертируется только одно указанное поле (`active`, `accent` или `slide`), что гарантирует корректную работу React-рендеринга.

### 3.3 Визуальная высота шагов (Heuristic Scaling)

**Строки: 141**

Высота визуального элемента шага рассчитывается логически:
$$Height = \begin{cases} 100\% & \text{если } Accent \\ 60\% & \text{если } Active \\ 15\% & \text{в режиме ожидания} \end{cases}$$
Это позволяет мгновенно оценить динамику паттерна без считывания текстовых данных.
