# 📘 АУДИТ: LiveSession.tsx

## 1. Глобальное Описание

### Назначение
Компонент `LiveSession` (NEURAL LIVE PRO) представляет собой высокопроизводительную инструментальную панель для живых выступлений. Он объединяет в себе матрицу запуска клипов (Clip Launcher), профессиональный микшер, FX-пэды для мгновенных эффектов и XY-интерфейсы для управления пространственной обработкой. Это основной интерфейс для перформанса, где задержки и удобство доступа к критическим функциям (Mute, Solo, Scene Launch) имеют приоритет.

### Связи
- **Audio Store:** Управляет всеми аудио-параметрами, запуском пресетов (Snapshots), громкостью и мастер-функциями (Play/Stop/Panic).
- **Visual Store:** Передает FFT-данные для спектральной визуализации и управляет сменой видов (`EXIT LIVE`).
- **Tone.js Engine:** Подключается к FX-шинам для управления задержкой и фильтрацией.

## 2. Интерфейс и UI Элементы

### Матрица клипов (Clip Matrix)
**Строки: 69-115**
Позволяет запускать один из 4 снепшотов для каждого из 6 инструментов.
- **Индикация квантования:** Показывает время до начала следующего такта (`NEXT BAR`), когда произойдет фактическое переключение сцены.
- **Scene Launchers:** Кнопки для одновременного запуска соответствующих снепшотов на всех дорожках.

### Pro Mixer
**Строки: 118-149**
Вертикальные ячейки управления (Strips) для каждого инструмента:
- **M (Mute) / S (Solo):** Мгновенное управление каналами.
- **Pro Faders:** Ползунки громкости с цветовой индикацией, соответствующей инструменту.

### Performance FX & XY Pads
**Строки: 152-184**
- **FX Pads:** Кнопки для временных эффектов (Tape Stop, Glitch, Noise), активные только при нажатии (Momentary).
- **Dual XY Pads:** Две сенсорные области для управления Delay (Space) и Distortion/Filter (Tone).

### Spectral Vision
**Строки: 188-205**
32-полосный анализатор спектра, отображающий текущий выходной аудиосигнал в реальном времени.

## 3. Логика и Математика

### 3.1 Управление XY-панелями

**Строки: 18-35**

Координаты касания нормализуются в диапазон $[0, 1]$ относительно границ элемента:
$$X = \text{clamp}(0, \frac{E.clientX - Rect.left}{Rect.width}, 1)$$
$$Y = \text{clamp}(0, 1 - \frac{E.clientY - Rect.top}{Rect.height}, 1)$$
Полученные значения напрямую маппятся на аудио-параметры (например, $X \times 0.5$ секунд для задержки).

### 3.2 Квантованный запуск (Clip Triggering)

**Строки: 73-75, 82-95**

Система использует `queuedSnapshots` для визуализации намерения переключить клип. Фактическое переключение происходит на уровне `audioStore` синхронно с сеткой тактов.

### 3.3 Спектральная визуализация

**Строки: 191-203**

FFT данные $(-100 \dots 0 \text{ dB})$ масштабируются для отображения высоты баров спектрограммы:
$$Height = \max(5\%, (Value_{dB} + 110) \times 0.8\%)$$
Это смещение на +110 гарантирует видимость баров даже при низком уровне сигнала (тишине).
