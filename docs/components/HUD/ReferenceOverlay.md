# 📘 АУДИТ: ReferenceOverlay.tsx

## 1. Глобальное Описание

### Назначение
Этот компонент представляет собой контекстный справочник (Reference HUD), который меняет свое содержимое в зависимости от того, что делает пользователь в данный момент. Если пользователь находится в режиме аранжировки — справочник показывает горячие клавиши для работы с клипами. Если сфокусирован на конкретном инструменте — справочник объясняет принципы работы синтеза этого инструмента (например, эвклидовы ритмы для барабанов или гранулярный синтез для семплера).

### Связи
- **Visual Store:** Считывает `appView`, `focusInstrument` и флаг видимости `showHelp`.
- **UI Architecture:** Является высокоуровневым оверлеем, не перекрывающим основной рабочий процесс, а дополняющим его.

---

## 2. Интерфейс и UI Элементы

### Контекстная логика (Content Provider)
Компонент динамически генерирует контент на основе текущего состояния приложения:
1. **Arrangement View:** Фокус на инструментах редактирования (Scissors, Duplicate, Automation).
2. **Drums Focus:** Описание алгоритма Бьёрклунда и эмуляций TR-808/909.
3. **Bass Focus:** Разница между Acid и FM движками.
4. **Sampler Focus:** Параметры гранулярного синтеза (Grain Size, Overlap).
5. **Overview (По умолчанию):** Глобальная 3D навигация и системные команды.

## 3. Логика и Математика

### 3.1 Фильтрация контента (Decision Tree)

**Строки:** 14-130

Внутри функции `getContent` реализовано дерево принятия решений. Оно приоритизирует текущий режим отображения (`appView`) над выбранным инструментом, что гарантирует актуальность справки для текущего контекста работы.

### 3.2 Визуальный макет

Справочник использует стилистику "стеклянного интерфейса" (`glass`). Каждая секция включает в себя:
- **Title & Icon:** Цветовое кодирование иконки совпадает с цветовой схемой инструмента в 3D.
- **Hints (Kbd):** Табличные данные с горячими клавишами и кратким описанием функций.
- **Footer:** Дополнительная техническая информация или советы.
