# 📘 АУДИТ: ArrangementEditor.tsx

## 1. Глобальное Описание

### Назначение
Компонент `ArrangementEditor` является самым сложным и функционально насыщенным модулем приложения. Это полноценный нелинейный редактор (Timeline), аналогичный профессиональным DAW (Ableton Live, Logic Pro). Он позволяет пользователю выстраивать музыкальную композицию, располагая "клипы" (состояния инструментов) во времени, управлять автоматизацией параметров, настраивать микшер для каждого трека и использовать профессиональные инструменты редактирования (ножницы, дублирование, выделение областью).

### Связи
- **Arrangement Store:** Центральное хранилище данных о клипах, треках, маркерах и автоматизации.
- **Audio Store:** Управляет глобальным транспортом (Play/Stop/BPM) через `Tone.js`.
- **Theme Store:** Динамически меняет визуальное оформление редактора (цвета, шрифты).
- **Snapshot Library:** Библиотека пресетов инструментов, которые и составляют содержимое клипов.

## 2. Интерфейс и UI Элементы

### Таймлайн и Дорожки (Lanes)
**Строки: 407-487**
Основное рабочее поле, разделенное на горизонтальные дорожки для каждого инструмента (Drums, Bass, Lead и т.д.).
- **ArrangementClipItem:** Визуальный блок клипа. Отображает превью содержимого (ритмические точки для барабанов или волны для синтезаторов).
- **Ruler (Линейка):** Отображает такты и доли. Поддерживает установку маркеров (Markers) и циклическое воспроизведение (Looping).

### Панель инструментов (Transport Pro)
**Строки: 321-384**
Верхняя панель с кнопками управления:
- **Snap Selection:** Привязка к сетке (Bar, 1/4, 1/16 и т.д.).
- **Scissors Tool:** Режим разрезания клипов.
- **Theme Selector:** Быстрая смена визуального стиля DAW.

### Инспектор клипа (Clip Inspector)
**Строки: 489-530**
Боковая панель, появляющаяся при выборе клипа. Позволяет точно настроить его ID, имя, громкость (Gain) и привязанный снепшот из библиотеки.

## 3. Логика и Математика

### 3.1 Расчет позиции (Tick-to-Pixel Mapping)

**Строки: 118, 166-187**

Все координаты в редакторе рассчитываются на основе тиков (Ticks) и уровня масштабирования (TPP — Ticks Per Pixel):
$$X_{position} = Tick_{start} \times TPP$$
$$Width = Tick_{duration} \times TPP$$
Для визуализации используется `requestAnimationFrame`, который перемещает `playhead` (курсор воспроизведения) синхронно с `Tone.Transport.ticks`.

### 3.2 Система привязки к сетке (Snapping)

**Строки: 229-236**

Математическая функция для округления координат мыши до ближайшего шага сетки:
$$SnappedTick = \lfloor \frac{Pixels_{mouse} / TPP}{SnapStep} \rfloor \times SnapStep$$
Где `SnapStep` зависит от выбранного разрешения (1n, 4n, 16n).

### 3.3 Автоматизация (Bezier Patterns)

**Строки: 453-470**

Автоматизация отрисовывается через SVG-пути. Каждая точка — это пара `[Tick, Value]`. Значение нормализовано (0..1) и преобразуется в высоту дорожки (80px):
$$Y_{svg} = (1 - Value) \times 80$$

### 3.4 Выделение областью (Marquee Selection)

**Строки: 288-307**

Сложная геометрическая проверка пересечения прямоугольника выделения и прямоугольников клипов для массового выбора объектов.

### Глубокий разбор логики и математики
- **Логика:** При дропе файла аудио-контекст декодирует данные, затем выполняется "Peak Detection" — сканирование массива сэмплов для нахождения максимальной амплитуды в каждом из 100 сегментов.
- **Математика:** Алгоритм расчета пиков:
  $$Peak_i = \max(|Sample_j|) \text{ for } j \in [i \times Segment, (i+1) \times Segment]$$
  Отрисовка в SVG через `M (i) (50-p) L (i) (50+p)`, создающая симметричную волну.
