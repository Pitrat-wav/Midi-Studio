# 📘 АУДИТ: CMISamplerView.tsx

## 1. Глобальное Описание

### Назначение
Этот компонент является цифровой репликой легендарной рабочей станции Fairlight CMI (Computer Musical Instrument). Он реализует интерфейс "Page R" — первый в мире графический секвенсор и семплер. Основная задача модуля — обеспечить управление гранулярным синтезатором, позволяя пользователю нарезать аудио-семплы на части (слайсы), изменять параметры "зерна" (grain) и программировать паттерны в стиле ранних 80-х.

### Связи
- **Sampler Store:** Хранит параметры гранулярного движка, сетку секвенсора и библиотеку семплов.
- **Audio Store:** Синхронизирует `bpm` и состояние воспроизведения для визуализации "бегущего огня".
- **AudioVisual Bridge:** Передает FFT-данные для отрисовки векторного осциллографа.

## 2. Интерфейс и UI Элементы

### Page R Grid (Сетка секвенсора)
**Строки: 88-104**
Классическая сетка 8x16. В текущей реализации активна первая дорожка для триггеринга семпла. Включает визуализацию текущего шага (`active-step`), синхронизированную с глобальным темпом.

### Векторный осциллограф (Canvas)
**Строки: 43-75, 108-113**
Реальное визуальное отображение звуковой волны (Spectrum) на 2D холсте. Отрисовывается зеленым цветом, имитируя люминофор старых ЭЛТ-мониторов.

### Панель управления (Sidebar)
**Строки: 127-177**
- **Библиотека семплов:** Список доступных аудиофайлов с навигацией "Вперед/Назад".
- **Параметры семпла:** Громкость, скорость воспроизведения (Rate) и количество слайсов.
- **Гранулярный движок:** Тонкая настройка размера зерна (Grain Size), нахлёста (Overlap) и детюна (Detune).

## 3. Логика и Математика

### 3.1 Расчет активного шага (Visual Synchronization)

**Строки: 25-40**

Для визуализации "бегущего курсора" без задержек используется `requestAnimationFrame` и расчет текущего шага на основе BPM:
$$StepDuration = \frac{60}{BPM} \div 4 \quad (\text{для 16-х нот})$$
$$CurrentStep = \lfloor \frac{PerformanceNow}{StepDuration} \rfloor \pmod{16}$$

### 3.2 Отрисовка векторного спектра

**Строки: 50-71**

FFT данные (массив из 64 значений) преобразуются в координаты Y на холсте:
$$Y = \frac{Height}{2} - \frac{Data_i}{255} \times \frac{Height}{2}$$
Линии соединяются методом `lineTo`, создавая непрерывный график амплитуды частот.

### 3.3 Гранулярные вычисления

Параметры `grainSize` (в секундах) и `overlap` (коэффициент перекрытия) напрямую влияют на то, как `Tone.GrainPlayer` извлекает фрагменты из аудиозаписи. Изменение `playbackRate` меняет скорость прохода по семплу без изменения его высоты (если включен гранулярный режим).
