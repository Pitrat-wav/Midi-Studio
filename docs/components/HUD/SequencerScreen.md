# 📘 АУДИТ: SequencerScreen.tsx

## 1. Глобальное Описание

### Назначение
Этот компонент является наиболее сложным логическим интерфейсом в приложении, объединяющим в себе функции классического модульного секвенсора ML-185 и вероятностного генератора "Turing Machine" (на базе Music Thing Modular). Он позволяет создавать бесконечные, самоизменяющиеся музыкальные последовательности, балансируя между строгим повторением и контролируемым хаосом.

### Связи
- **Sequencer Store:** Хранит состояния всех "стейджей" (этапов), регистры машины Тьюринга и параметры змейковидных паттернов.
- **Harmony Store:** Принудительно квантует выходные ноты секвенсора согласно выбранной тональности и ладу.
- **Visual Store:** Обеспечивает выход из оверлея.

## 2. Интерфейс и UI Элементы

### Редактор этапов (Stage Sequencer)
**Строки: 46-93**
Каждый этап (Stage) имеет три ключевых параметра:
- **LEN (Length):** Количество шагов, которое продлится этот этап.
- **PLS (Pulses):** Количество импульсов (ударов), которые прозвучат внутри этапа.
- **PITCH:** Базовая высота ноты этапа.

### Машина Тьюринга (Turing Machine)
**Строки: 96-114**
Визуализирует работу сдвигового регистра (Shift Register). 
- **Binary Display:** Отображает текущее состояние битов в регистре.
- **Probability:** Вероятность инверсии бита при каждом шаге, что создает новые музыкальные вариации.

### Snake Pattern
**Строки: 117-119**
Отображает выбранный алгоритм обхода сетки секвенсора (например, Пинг-Понг, Спираль и т.д.).

## 3. Логика и Математика

### 3.1 Алгоритм сдвигового регистра (Turing Logic)

**Строки: 101-109**

Генерация последовательности основана на циклическом сдвиге числа в памяти:
$$NextRegister = (Register \ll 1) \mid (\text{Random} < Probability ? \neg \text{LastBit} : \text{LastBit})$$
Визуализация в компоненте преобразует десятичное число в двоичную строку заданной длины (`turingBits`) и отрисовывает её как ряд индикаторов.

### 3.2 Комбинированный секвенсинг

Система позволяет накладывать эвклидовы ритмы (`pulseCount`) на временные интервалы (`length`) каждого этапа. Это создает полиритмические структуры, где каждый шаг секвенсора может иметь разную длительность и внутренний ритм.

### 3.3 Квантование (Musical Scale Constraint)

Хотя в интерфейсе задается прямой MIDI Pitch (0-127), логика в сторе (связанная с `harmonyStore`) автоматически подтягивает эти значения к ближайшим нотам выбранного лада, предотвращая диссонансы.
