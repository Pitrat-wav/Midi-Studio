# 📘 АУДИТ: WebGLScene.tsx

## 1. Глобальное Описание

### Назначение
Этот компонент является главным 3D-контейнером приложения. Он инициализирует `Canvas` (холст) для отрисовки WebGL-графики через библиотеку `@react-three/fiber`. **WebGLScene** объединяет в себе все визуальные элементы (инструменты, фоны, визуализаторы), управляет камерой и обрабатывает сложные жесты управления (например, радиальное меню и жесты по краям экрана).

### Связи
- **R3F Ecosystem:** Координирует работу всех 3D-компонентов в папке `src/components/WebGL/`.
- **Gesture Manager:** Использует `useGestureStore` для управления интерактивностью.
- **Audio Engine:** Интегрирован с `audioStore` для синхронизации визуальных эффектов с темпом и громкостью.
- **Hand Tracking:** Инициализирует хук `useHandTracking` для AI-управления.

## 2. Интерфейс и UI Элементы

### 3D Контент
- **AllInstruments3D:** Рендеринг всех музыкальных модулей в пространстве.

### Жестовое управление
**Строки:** 67-82
- **Long Press:** Вызов радиального меню для переключения режимов (Focus, Overview).
- **Edge Swipe (Top):** Управление общей громкостью (Master Volume).
- **Edge Swipe (Bottom):** Управление темпом (BPM).

## 3. Логика и Математика

### 3.1 Обработка жестов по краям экрана

**Строки:** 74-80

Для плавного изменения параметров звука используются экранные координаты курсора:
**Математика громкости (Top Edge):**
$$Volume = \text{clamp}(1 - \frac{Y_{coord}}{200}, 0, 1)$$
Где 200 — зона чувствительности в пикселях от верхнего края.

**Математика темпа (Bottom Edge):**
$$BPM = \text{clamp}(60 + (Height_{window} - Y_{coord}) \times 0.5, 60, 240)$$
Это позволяет менять темп от 60 до 240 BPM физическим движением пальца.

### 3.2 Камера и Фокус

**Строки:** 126-129

Компонент `CameraController` получает инструкции от `WebGLScene`. Если `focusInstrument` не равен `null`, камера автоматически переходит в режим `focus`, плавно подлетая к выбранному инструменту.
