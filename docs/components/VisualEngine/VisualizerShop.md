# VisualizerShop.tsx

### 1. Глобальное Описание (File Overview)
* **Назначение:** Галерея ("Магазин") для выбора визуальных режимов. Предоставляет поиск, фильтрацию по категориям и предпросмотр доступных шейдеров/сцен.
* **Связи:** 
    * Импортирует: `useVisualStore`, `VISUALIZER_REGISTRY`.
    * Используется в: `App.tsx` (глобальный оверлей).
    * Стили: `VisualizerShop.css`.

### 2. Интерфейс и UI Элементы (Visual Elements)

* **Элемент:** Поиск (Input)
    * **Где находится:** Строки 80-86.
    * **Логика:** Фильтрует список визуализаторов по имени и тегам.
    * **Связанные функции:** `setSearchQuery`.

* **Элемент:** Табы категорий (Buttons)
    * **Где находится:** Строки 91-100.
    * **Логика:** Переключает фильтр `activeCategory` (ALL, 2D, 3D, AUDIO, VIDEO). Отображает счетчик доступных элементов.

* **Элемент:** Карточка визуализатора (Card)
    * **Где находится:** Строки 106-126.
    * **Логика:**
        * **Click:** Выбирает визуализатор (`handleSelect`) и закрывает магазин.
        * **Drag:** Позволяет перетащить (Drag&Drop) ID визуализатора (для будущих фич).
        * **Haptic:** Триггерит 'light' вибрацию на мобильных устройствах при начале перетаскивания.

### 3. Логика и Математика (Logic & Math)

* **Функция:** `categories` (useMemo)
* **Что делает:** Заранее распределяет `VISUALIZER_REGISTRY` по группам на основе тегов.
    * `AUDIO`: теги spectrum, pulse, audio...
    * `VIDEO`: теги video, cam, feedback...
    * `2D` / `3D`: наличие/отсутствие тега `2d`.

* **Функция:** `filteredVisualizers` (useMemo)
* **Что делает:** Применяет два фильтра последовательно:
    1. По категории (`categories[activeCategory]`).
    2. По поисковому запросу (`includes(searchQuery)`).

* **Математическая модель (Filtering):**
    $$ Result = \{ v \in Registry \mid v.category = ActiveCategory \land (v.name \supset Query \lor v.tags \supset Query) \} $$
