# 📘 АУДИТ: Keyboard3D.tsx

## 1. Глобальное Описание

### Назначение
Компонент `Keyboard3D` является интерактивным музыкальным контроллером в 3D-пространстве. Это виртуальная клавиатура (синтезатор), которая позволяет пользователю исполнять мелодии в реальном времени, взаимодействуя с 3D-моделью клавиш. Дизайн выполнен в стиле премиальных синтезаторов с деревянными боковыми панелями, светодиодной подсветкой и колесами модуляции.

### Связи
- **Audio Store:** Передает команды `triggerNote` в ведущий синтезатор (`leadSynth`) при нажатии на клавиши.
- **AudioVisual Bridge:** Получает данные о пульсации звука для динамической подсветки клавиш и колес модуляции.
- **Telegram WebApp:** Интеграция с системой тактильной отдачи (Haptic Feedback) для повышения физического ощущения от игры.

## 2. Интерфейс и UI Элементы

### Структура клавиш (Key3D)
**Строки: 41-108**
- **Белые и черные клавиши:** Реализованы через `RoundedBox` с корректными пропорциями и материалами.
- **Интерактивность:** При нажатии клавиша опускается вниз по оси Y и наклоняется, имитируя физический механизм.
- **Свечение:** Каждая нажатая клавиша подсвечивается бирюзовым цветом (`#00ffcc`).

### Органы управления перформансом (Performance Wheels)
**Строки: 14-39, 170-173**
Два колеса (Pitch и Mod) в левой части корпуса. Колесо Pitch визуально отклоняется при взаимодействии, а колесо Mod пульсирует в такт исполняемой музыке.

### Корпус и Декор (Chassis)
**Строки: 149-190**
- **Premium Finish:** Металлический корпус с текстурированными деревянными боковинами (`#4a2c16`).
- **Branding & LED:** Светящаяся надпись "TELEGRAM MIDI STUDIO PRO" и LED-полоса вдоль всей клавиатуры.

## 3. Логика и Математика

### 3.1 Расчет раскладки (Keyboard Layout)

**Строки: 114-139**

Алгоритм автоматически вычисляет координаты $[x, y, z]$ для двух октав клавиш:
- Ширина белой клавиши: $0.82$ ед.
- Позиционирование черных клавиш: Смещаются на $-0.41$ по X относительно белой и поднимаются на $+0.12$ по Y.
- Это гарантирует математически точную геометрию стандартной клавиатуры пианино.

### 3.2 Обработка нажатий и Velocity

**Строки: 74-84**

Сила нажатия (Velocity) определяется по позиции клика на клавише:
$$Velocity = 0.3 + UV_{y} \times 0.7$$
Где $UV_y = 1.0$ (край клавиши) дает максимальную громкость, а $UV_y = 0.0$ (ближе к корпусу) — минимальную. Это имитирует динамику реальной игры.

### 3.3 Инерция и сглаживание (Lerp)

**Строки: 70-71**

Движение клавиш при нажатии и отпускании сглаживается с помощью линейной интерполяции:
$$Pos = \text{lerp}(Pos_{current}, Pos_{target}, 0.3)$$
Это предотвращает мгновенный "телепортационный" переход и придает анимации вес.
