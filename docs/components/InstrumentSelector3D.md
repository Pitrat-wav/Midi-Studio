# 📘 АУДИТ: InstrumentSelector3D.tsx

## 1. Глобальное Описание

### Назначение
Этот компонент реализует плавающий трехмерный интерфейс выбора инструментов (Navigation HUD), который всегда находится перед глазами пользователя в нижней части экрана, независимо от положения камеры. Он позволяет быстро переключаться между барабанами, басом, пэдами и другими модулями, используя интуитивно понятные круглые кнопки с текстовыми метками.

### Связи
- **Spatial Layout:** Переключает фокус камеры через `onSelect`.
- **Three.js / R3F:** Использует `camera.quaternion` для того, чтобы меню "следовало" за взглядом пользователя.
- **Visual Store:** Отображает текущий активный инструмент.

---

## 2. Интерфейс и UI Элементы

### Структура меню
- **Radial Layout:** Кнопки инструментов расположены по дуге (арке), что создает эффект футуристичного интерфейса кабины пилота.
- **InstrumentButton:** Индивидуальный круглый элемент для каждого инструмента:
  - **Scale Feedback:** Кнопка увеличивается при наведении или активации.
  - **Rotation:** Активная кнопка плавно вращается вокруг своей оси.
  - **Text Label:** Название инструмента, отрисованное с помощью `Text` (drei), с контрастной обводкой для читаемости.
  - **Active Ring:** Светящееся кольцо вокруг активного инструмента.

## 3. Логика и Математика

### 3.1 Следование за камерой (Relative Positioning)

**Строки:** 119-131

Чтобы выборщик всегда был виден, его позиция рассчитывается в локальном пространстве камеры в каждом кадре:
**Математика смещения:**
$$\vec{Offset} = (0, -1.8, -4.0)$$
$$\vec{WorldPos} = \vec{CamPos} + (\vec{Offset} \times \text{Quaternion}_{Cam})$$
Это удерживает меню на фиксированном расстоянии (4 единицы) и высоте (-1.8 единиц) относительно взгляда.

### 3.2 Расположение по дуге (Arc Arrangement)

**Строки:** 133-145

Кнопки распределяются по окружности:
$$\text{Angle}_i = \text{StartAngle} + (i + 1) \times \frac{\pi}{N+1}$$
$$X = \cos(\text{Angle}_i) \times \text{Radius}$$
$$Y = \sin(\text{Angle}_i) \times \text{Radius}$$
Где $N$ — количество инструментов. Это создает гармоничную арку вокруг центральной кнопки "Overview".

### 3.3 Интерактивность

При наведении курсора на кнопку срабатывает `onPointerEnter`, который меняет системный курсор на `pointer`. При клике вызывается `onSelect`, инициируя поворот всей 3D-сцены к выбранному модулю.
