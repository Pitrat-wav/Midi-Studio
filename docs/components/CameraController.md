# 📘 АУДИТ: CameraController.tsx

## 1. Глобальное Описание

### Назначение
Этот компонент управляет "глазами" пользователя в 3D пространстве. Он обеспечивает плавные кинематографические переходы между инструментами, поддерживает ручное управление полетом (Fly Mode) с клавиатуры или геймпада, а также интегрирует стандартную систему вращения `OrbitControls`. Это критически важный узел, определяющий, как пользователь воспринимает пространственную архитектуру студии.

### Связи
- **Spatial Layout:** Содержит координаты и углы обзора (`cameraPreset`) для каждого инструмента.
- **Visual Store:** Считывает и обновляет фокус-инструмент.
- **Gamepad Manager:** Получает данные от стиков контроллеров (Xbox, DualSense) для навигации.
- **Gestures:** Взаимодействует с системой жестов для предотвращения конфликтов управления при работе с ручками (Knobs).

## 2. Интерфейс и UI Элементы

### Режимы камеры
1. **Overview (Общий обзор):** Камера находится на удалении, показывая все орбитальное кольцо инструментов.
2. **Focus (Фокус):** Камера подлетает вплотную к выбранному модулю, обеспечивая удобный доступ к его элементам управления.
3. **Manual Fly (Ручной полет):** Свободное перемещение в пространстве с помощью WASD или стиков геймпада.

## 3. Логика и Математика

### 3.1 Плавная интерполяция (Lerp)

**Строки:** 88-99

Для создания эффекта "подлета" используется линейная интерполяция векторов позиции камеры и точки взгляда (Target).
**Математика перехода:**
$$P_{new} = P_{current} + (P_{target} - P_{current}) \times (5 \times \Delta t)$$
$$Look_{new} = Look_{current} + (Look_{target} - Look_{current}) \times (5 \times \Delta t)$$
Где $\Delta t$ — время кадра, а 5 — коэффициент скорости полета.

### 3.2 Векторная навигация (Fly Mode)

**Строки:** 124-171

При ручном полете направление движения рассчитывается относительно направления взгляда камеры:
1. Берется вектор направления камеры $Forward$.
2. Проецируется на плоскость горизонта $(Forward.y = 0)$.
3. Вычисляется боковой вектор $Right = Forward \times Up$.
4. Позиция камеры и точка цели смещаются синхронно:
   $$Pos = Pos + (Forward \times Speed_{y}) + (Right \times Speed_{x})$$

### 3.3 Управление с геймпада

**Строки:** 152-165

Используются данные от аналоговых стиков:
- **Левый стик:** Управление перемещением по осям X и Y.
- **Правый стик (X):** Вращение камеры вокруг центра (yaw).
Математика учитывает "мертвую зону" (deadzone) 0.1 для исключения дрейфа камеры.
