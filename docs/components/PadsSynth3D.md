# 📘 АУДИТ: PadsSynth3D.tsx

## 1. Глобальное Описание

### Назначение
Компонент `PadsSynth3D` визуализирует генеративный эмбиент-синтезатор через метафору "облака частиц". Он преобразует абстрактные параметры звука (яркость и сложность) в физические свойства 3D-объекта в реальном времени. Инструмент предназначен для создания атмосферных подкладов и визуализации гармонического пространства студии.

### Связи
- **Pad Store:** Источник параметров `brightness`, `complexity` и состояния `active`.
- **Gesture Manager:** Позволяет управлять параметрами звука, напрямую "перетаскивая" облако частиц в 3D.
- **Spatial Layout:** Определяет положение облака и регуляторов в пространстве.

## 2. Интерфейс и UI Элементы

### Облако частиц (Particle Cloud)
**Строки: 91-121**
- **Количество:** Динамически меняется от 500 до 2000 частиц в зависимости от сложности (`complexity`).
- **Распределение:** Сферическое разреженное облако, создающее ощущение объема и тумана.
- **Свечение:** Используется `AdditiveBlending`, что заставляет частицы накладываться друг на друга, создавая яркие участки в центрах скоплений.

### Интерактивные 3D-регуляторы
**Строки: 124-154**
- **Brightness Knob:** Оранжевый регулятор для управления тембром.
- **Complexity Knob:** Регулятор для изменения плотности гармоник и количества частиц.
- **Power Button:** Кнопка включения/выключения атмосферы.

## 3. Логика и Математика

### 3.1 Генерация облака (Spherical Distribution)

**Строки: 29-52**

Частицы распределяются внутри сферы случайным образом, используя сферические координаты:
$$X = Radius \times \sin(\phi) \times \cos(\theta)$$
$$Y = Radius \times \sin(\phi) \times \sin(\theta)$$
$$Z = Radius \times \cos(\phi)$$
Где $Radius \in [1, 3]$, $\theta \in [0, 2\pi]$, а $\phi$ рассчитывается через арккосинус для равномерного распределения по поверхности.

### 3.2 Температурная карта яркости (Color Mapping)

**Строки: 44-45**

Цвет частиц зависит от параметра `brightness`. Он мапится на цветовой круг (Hue):
$$Hue = \text{lerp}(0.55_{cyan}, 0.15_{orange}, Brightness)$$
Это создает интуитивную связь: "холодный" звук — сине-голубой, "теплый/яркий" — оранжево-золотой.

### 3.3 Пространственное управление жестами

**Строки: 64-72**

Пользователь может менять параметры, перетаскивая облако. Движение по горизонтали (dx) меняет `complexity`, по вертикали (dy) — `brightness`. Это превращает синтезатор в "музыкальную плазму", которой можно манипулировать в пространстве.

### 3.4 Динамическая пульсация

**Строки: 75-80**

При активации синтезатора облако начинает "дышать" по синусоидальному закону:
$$Pulse = \sin(Time \times 2) \times 0.1 + 1$$
Масштаб облака интерполируется (`lerp`) к этому значению, создавая живую пульсацию.
