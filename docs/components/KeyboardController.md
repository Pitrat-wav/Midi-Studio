# 📘 АУДИТ: KeyboardController.tsx

## 1. Глобальное Описание

### Назначение
Этот компонент является глобальным диспетчером горячих клавиш (Shortcut Manager) приложения. Он перехватывает нажатия клавиш на уровне окна и преобразует их в команды для аудио-движка, интерфейса и 3D-навигации. Компонент не имеет визуального представления (`returns null`), работая как фоновый слушатель событий.

### Связи
- **Store:** Взаимодействует напрямую с `audioStore`, `visualStore`, `bassStore`, `drumStore` и `padStore`.
- **Infrastructure:** Реализован с использованием `useEffect` для добавления и удаления слушателей событий `keydown`.
- **Performance:** Использует паттерн "прямого доступа" (`getState()`) внутри обработчика события, чтобы избежать лишних ререндеров компонента при изменении глобального состояния.

## 2. Логика и Математика

### 2.1 Маппинг горячих клавиш

Компонент реализует следующие группы команд:

#### Транспорт и системные вызовы
- **Space:** `audio.togglePlay()` — Старт/Стоп проигрывания.
- **Key P (Panic):** `audio.panic()` — Мгновенное прекращение всех звуков (сброс MIDI-голосов).
- **Key H:** Переключение системного оверлея.
- **Tab:** Переключение между 3D и 2D видами (`visual.cycleView()`).

#### Навигация по инструментам (Digit Keys)
Клавиши **1-9** осуществляют быстрый переход фокуса камеры и HUD к конкретному инструменту:
1. Drums
2. Bass
3. Harmony
4. Pads
5. Sequencer
6. Drone
7. Master
8. Sampler
9. Buchla
0. Сброс фокуса (Общий вид).

#### Управление звуком
- **Key M (Mute):** Заглушает выбранный в данный момент инструмент. Использует карту соответствия (`channelMap`) между UI-фокусом и аудио-каналами.
- **[ / ] (Brackets):** Регулировка мастер-громкости с шагом 0.1 Gain.

---

### 2.2 Математика регулировки громкости

**Строки:** 121-132

Регулировка громкости происходит на уровне `Tone.Destination`.
$$Volume_{new} = \text{clamp}(Tone.dbToGain(CurrentDb) \pm 0.1, 0, 1)$$

Так как `Tone.js` работает с децибелами (логарифмическая шкала), а интерфейс с линейным коэффициентом усиления (Gain), происходит конвертация для обеспечения интуитивно понятного изменения громкости на 10% за нажатие.

---

### 2.3 Фильтрация ввода

**Строки:** 23-26

Чтобы горячие клавиши не срабатывали, когда пользователь печатает в полях ввода (например, в чате или при именовании пресета), реализована проверка:
```javascript
if (target.tagName === 'INPUT' || target.tagName === 'TEXTAREA') return
```
