# 📘 АУДИТ: Mixer3D.tsx

## 1. Глобальное Описание

### Назначение
Этот компонент представляет собой физическую 3D-модель профессиональной микшерной консоли внутри виртуальной студии. Он служит интерфейсом для управления уровнями громкости и состоянием Mute каждого из 5 основных аудио-каналов. Дизайн консоли выполнен в стиле премиального аудио-оборудования с использованием высококачественных материалов и световой индикации.

### Связи
- **Audio Store:** Основной источник данных о громкости и состоянии Mute для всех каналов. Также вызывает функцию `panic()` через мастер-кнопку.
- **Spatial Layout:** Определяет положение консоли в 3D-пространстве студии.
- **Knob3D:** Переиспользуемый 3D-контроллер, адаптированный здесь для работы в качестве фейдера.

## 2. Интерфейс и UI Элементы

### Канальные линейки (Fader3D)
**Строки: 21-63**
Каждая линейка канала содержит:
- **Track (Дорожка):** Вертикальное углубление, по которому перемещается регулятор.
- **Fader (Фейдер):** 3D-манипулятор, чья позиция по вертикали (Y) соответствует значению громкости.
- **Mute Button:** Кнопка с подсветкой. Если канал заглушен, кнопка горит ярко-красным цветом.
- **Dynamic Text:** Текстовые метки канала (DRM, BASS, LEAD и др.) с цветовой кодировкой.

### Консоль и Мастер-секция
**Строки: 65-111**
- **Console Base:** Основание из темного шлифованного металла (`WhiskMaterial`).
- **Master Panic Button:** Большая красная цилиндрическая кнопка в нижней части консоли для мгновенной остановки всего звука в случае системной ошибки или перегрузки.

## 3. Логика и Математика

### 3.1 Расчет позиции фейдера

**Строки: 40-48**

Позиция регулятора `Knob3D` на вертикальной оси рассчитывается как линейное преобразование нормализованного значения громкости $[0, 1]$ в диапазон перемещения $[-1.75, 1.75]$ в 3D единицах:
$$Y_{fader} = (Value_{vol} - 0.5) \times 3.5$$
Это позволяет использовать стандартную логику перемещения ручки, сохраняя при этом вид профессиональной микшерной консоли.

### 3.2 Управление состоянием (Mute Logic)

При изменении состояния `isMuted` происходит пересчет цвета регулятора. Если канал выключен, регулятор становится серым (`#444`), что дает мгновенную визуальную обратную связь, помимо светящейся кнопки Mute.

### 3.3 Взаимодействие (Pick & Interact)

Используются нативные события `react-three-fiber` (`onClick`, `onPointerMove`). Важное дополнение — `e.stopPropagation()`, которое предотвращает "прокликивание" сквозь микшер на объекты, находящиеся за ним (например, планеты фона).
