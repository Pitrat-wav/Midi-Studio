# 3.1 MIDI Protocol & Export

The MIDI Studio generates standard MIDI file format 0/1 data from the generative sequencers.

## Track Mapping
Each instrument in the studio is assigned to a specific MIDI channel and CC range:

| Instrument | MIDI Channel | CC Range | Note Range |
|------------|--------------|----------|------------|
| TR-808     | 10           | N/A      | C1 - G#1   |
| TR-909     | 10           | N/A      | C1 - G#1   |
| HarmSynth  | 1            | 70-85    | C2 - C6    |
| Sampler    | 2            | 10-20    | C3 - C4    |

## Export Logic
- **Real-time Capture**: The `MIDIExportManager.ts` (located in `src/lib/`) captures MIDI events from the `AudioStore` dispatcher.
- **Smart Chord Integration**: Notes generated by the Harmony module are quantized to the global scale before being written to the MIDI buffer.
- **File Generation**: Uses the `midi-writer-js` library to compile the buffer into a `.mid` file for user download via the Telegram WebApp interface.

## Limitations
- **Polyphony**: Currently limited to 8 voices per hardware track.
- **Automation**: CC automation is exported for HarmSynth parameters but not for the analog-style drum knobs (which are internal to the engine).
