# Фронтенд (FRONTEND.md)

## Технологический стек
- **React 18**: Функциональные компоненты и хуки.
- **Vite**: Сборка и быстрый HMR.
- **Framer Motion**: Плавные анимации UI и переходы между табами.
- **Lucide React**: Набор иконок.

## Структура компонентов

### 1. Основные виды (Views)
- **`DrumsView`**: Интерфейс драм-машины. Использует CSS Grid для визуализации паттернов Bjorklund.
- **`BassView`**: Управление Acid и FM басом. Содержит кастомные регуляторы (Knobs) и осциллограф.
- **`SequencerView`**: Главный хаб для Lead и Poly инструментов. Включает **Chord Monitor** для визуализации гармонии.
- **`HarmonyView`**: Глобальные настройки лада и макро-контроллеры (Space, Energy).
- **`HelpView`**: Обучающий раздел с концептуальным описанием Smart Chord.

### 2. Общие компоненты
- **`Knob`**: Кастомный регулятор, оптимизированный для тач-интерфейсов. Поддерживает drag-жесты.
- **`Oscilloscope`**: Canvas-компонент, визуализирующий выходной сигнал Bass-синтезатора.
- **`TabNavigation`**: Нижняя панель навигации с поддержкой мобильных Safe Areas.

## Управление UI-состоянием
- **Zustand Selectors**: Используются для минимизации ререндеров. Компоненты подписываются только на нужные поля (напр., `isPlaying`).
- **Tone.Draw**: Все визуальные вспышки шагов в секвенсоре синхронизированы с аудио-событиями через планировщик Tone.js.

## Интеграция с Telegram
- Использование `window.Telegram.WebApp` для:
  - Установки цвета заголовка.
  - Закрытия приложения.
  - Отправки MIDI-файлов.
  - Адаптации под тему (Dark/Light).

## Оптимизация
- Динамический импорт не используется (размер бандла позволяет быструю загрузку).
- Пассивные слушатели событий для регуляторов обеспечивают плавность прокрутки на мобильных устройствах.
