# Telegram MIDI Studio - Документация для разработчика (Agent Docs)

## Технический обзор
Telegram MIDI Studio — это генеративная музыкальная среда, созданная для Telegram Mini Apps (TMA). Она сочетает в себе алгоритмический секвенсинг и аналоговый синтез звука.

### Структура проекта
- **Аудио и визуализация**:
  - `audioStore`: Управление глобальным движком, воркараунд для iOS и уровни микшера.
  - `Oscilloscope`: React-компонент для визуализации волновой формы в реальном времени.
  - `MixerView`: Контроль громкости всех внутренних инструментов.

- **Генеративная логика**:
  - `StingGenerator`: Детерминированная плотность (Density) с использованием псевдослучайных зеркал (seeds).
  - `DrumMachine`: Индивидуальные плееры для сэмплов 808/909 с контролем тона и затухания.
  - `GridWalker`: Статическая логика движения для сетки MDD Snake.
  - `PadGenerator`: Генерация полифонических аккордов на базе ладов Tonal.js.
- `/src/store`: Глобальное управление состоянием через Zustand.
  - `audioStore.ts`: Аудиоконтекст, управление транспортом и глобальный плейхед.
  - `instrumentStore.ts`: Параметры барабанов, баса и секвенсора.
- `/src/components`: React-компоненты (регуляторы, экраны, петля секвенсора).

### Реализация ключевых функций
1. **Sequencer Loop**: Цикл `Tone.Loop` (16-е ноты) управляет всем движком. Он обращается к хранилищу Zustand напрямую, что позволяет менять параметры в реальном времени без сбоев. Использует `Tone.Draw` для пофреймовой синхронизации плейхеда интерфейса.
2. **Логика ML-185**: Стадии (Stages) определяют длительность ноты (в пульсах). Режимы гейта определяют, триггерится ли нота один раз, повторно (ratchet) или удерживается (tie).
3. **MDD Snake**: Сетка 4x4, где мелодия определяется "прогулкой" по ячейкам. Паттерн движения (Linear, Spiral и т.д.) создает плавно меняющиеся мелодические фигуры.
4. **Экспорт**: Генерация MIDI через `midi-writer-js`, конвертация в Base64 и отправка на эндпоинт Bot API для доставки пользователю в чат.

### Примечания по обслуживанию
- **Audio Context**: Всегда требуется взаимодействие пользователя (кнопка Start) перед вызовом `Tone.start()`.
- **Производительность**: Избегайте тяжелых вычислений внутри колбэка `Tone.Loop`. Используйте предварительно рассчитанные паттерны, где это возможно.
- **Стилизация**: Используйте переменные `--tg-theme-*` для CSS, чтобы обеспечить нативный вид как в светлой, так и в темной темах Telegram.
